## 7. 아직 남은 리스크

- **community create/get_queryset** 에서 `request.tenant` 없을 때 `request_student.tenant` 사용 — “동일 사용자 소속 tenant”이지만 원칙상 fallback에 해당. resolver가 정상이면 `request.tenant`가 있으므로 실질 리스크는 낮음. 필요 시 403만 반환하도록 제거 가능.
- **원격 검증 미실행** — run-qna-e2e-verify.ps1 등은 코드만 점검했고, 실제 배포 후 스크립트 실행은 하지 않음. 필요 시 배포 후 수동 실행 권장.

---

## 8. 추가로 바로 실행해야 할 검증 시나리오

- 배포 반영 확인이 필요하면: `pwsh scripts/v1/deploy.ps1 -AwsProfile default -SkipBuild` 후 `pwsh scripts/v1/run-qna-e2e-verify.ps1 -AwsProfile default` 실행.
- tenant cross-check: 동일 tenant에서만 데이터 노출·다른 tenant에서 미노출은 기존 E2E(verify_qna_e2e)로 일부 검증됨. 추가 시나리오는 운영에서 필요 시 설계.
