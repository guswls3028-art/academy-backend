# 500 배포

**리전**: ap-northeast-2 (서울)  
**원칙**: Video Worker **4GB RAM + 100GB EBS** 필수. 8000 포트는 테스트용만 → 오픈 전 ALB+HTTPS.  
**워커**: ASG Target Tracking — 평소 0대, 큐 쌓이면 증설(Min=0, Max=20). [설계.md](설계.md) 참고.

## 0. 프론트엔드 배포

| 항목 | 내용 |
|------|------|
| **방식** | `git push origin main` → Cloudflare Pages 자동 배포 |
| **레포** | `frontend/` (별도 git repo) |
| **별도 스크립트 불필요** | `deploy-front.ps1`, `deploy.ps1 -DeployFront` 사용 금지 (params.yaml SSOT 값 미설정) |
| **확인** | Cloudflare Pages 대시보드에서 빌드 상태 확인 |

> 프론트엔드는 백엔드와 독립된 레포·배포 파이프라인이다. 백엔드 배포와 무관하게 `git push`만으로 반영된다.

## 1.1 배포 방식

| 방식 | 설명 | 권장 |
|------|------|------|
| **CI 자동 배포** | main push → GitHub Actions가 5개 이미지 빌드·ECR 푸시 → `deploy-api-refresh` job이 API ASG instance refresh 실행. **push=서버 반영** 자동화. | 일상적인 코드 반영. |
| **수동 정식 배포** | `scripts/v1/deploy.ps1` — 전체 인프라 Ensure + API LT 갱신 + ASG instance refresh. | 인프라 변경·안정 반영·릴리즈용. |

## 10. 로컬에서 배포 DB 사용

로컬에서 `python manage.py shell`, `manage.py migrate`, `fix_qna_orphan_created_by` 등을 실행할 때 **배포(프로덕션) RDS/Redis**를 쓰려면, SSM에 저장된 API env를 로컬 `.env`에 반영하면 됩니다.

| 목적 | 명령 (로컬 PowerShell) |
|------|-------------------------|
| **배포 DB/Redis를 .env에 반영** | `pwsh scripts/v1/sync-local-env-from-ssm.ps1 -AwsProfile default` |
| **반영 내용만 확인 (파일 수정 없음)** | `pwsh scripts/v1/sync-local-env-from-ssm.ps1 -AwsProfile default -WhatIf` |

- SSM `/academy/api/env`(JSON)에서 `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_PORT`, `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD`, `REDIS_DB` 만 읽어와 루트 `.env`에 덮어씁니다. 나머지 키는 그대로 둡니다.
- **주의**: 배포 DB를 쓰면 실제 데이터가 변경됩니다. `migrate`, `fix_qna_orphan_created_by` 등 실행 시 유의하세요.
