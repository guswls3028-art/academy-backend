# 테넌트 도메인 연결 — 가비아 네임서버 (Cloudflare 연결용)

**도메인을 Cloudflare(클플)에 연결하려면**, 가비아 도메인 관리에서 **네임서버를 Cloudflare에서 받은 값**으로 바꿔야 합니다.  
(가비아 기본 NS가 아니라 **클플에서 부여한 NS**를 등록해야 클플 연결됨.)

---

## 1. 가비아에 넣을 값 (Cloudflare 네임서버)

클플에 **sswe.co.kr**을 사이트로 추가한 뒤, 클플 대시보드에 표시되는 **네임서버 2개**를 그대로 가비아에 등록하면 됩니다.

### 현재 계정에서 쓰는 Cloudflare 네임서버 (참고)

다른 테넌트 도메인들이 같은 클플 계정에서 쓰는 NS 예시는 아래와 같습니다.  
**sswe.co.kr**은 클플에 추가한 뒤 표시되는 2개를 사용하세요.

| 도메인 | 1차 네임서버 | 2차 네임서버 |
|--------|----------------|----------------|
| hakwonplus.com, limglish.kr, tchul.com | `barbara.ns.cloudflare.com` | `thaddeus.ns.cloudflare.com` |
| ymath.co.kr | `malcolm.ns.cloudflare.com` | `zita.ns.cloudflare.com` |

- **sswe.co.kr**: 아직 Cloudflare에 없음 → **먼저 클플에서 사이트 추가** 후 나온 2개를 가비아에 등록.

---

## 2. 절차 (클플 연결까지)

1. **Cloudflare 대시보드** → [사이트 추가](https://dash.cloudflare.com/) → 도메인에 `sswe.co.kr` 입력.
2. 플랜 선택 후 **다음** → **"네임서버 변경"** 단계에서 **2개 네임서버**가 표시됨 (예: `xxx.ns.cloudflare.com`, `yyy.ns.cloudflare.com`).
3. **가비아** [도메인 관리](https://domain.gabia.com) → **sswe.co.kr** → **도메인 정보 변경** 또는 **네임서버 설정**.
4. 1차·2차에 클플에서 복사한 **2개 값을 그대로 붙여넣기** (3차는 비우거나 그대로 둠).
5. 저장 후 DNS 전파 대기 (수 분~최대 48시간).
6. 클플 대시보드에서 **DNS 검증 완료** 확인.

---

## 3. 나중에 네임서버 값 다시 확인하려면

클플에 sswe.co.kr 추가 후, 아래 스크립트로 zone별 네임서버를 다시 조회할 수 있습니다.

```powershell
cd backend
.\scripts\get-cloudflare-nameservers.ps1
```

`backend\.env`에 `CLOUDFLARE_EMAIL`, `CLOUDFLARE_API_KEY`가 있어야 합니다.

---

## 4. 적용 후 (앱/API 쪽)

- 코드/DB에 테넌트·도메인 추가: `python manage.py setup_three_tenants` (또는 관리 API).
- DNS 전파 후 해당 도메인으로 접속 시 해당 테넌트로 라우팅됩니다.
