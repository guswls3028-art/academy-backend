# V1.2.7 Release Notes

**Status:** production-deployed  
**Release date:** 2026-05-23 KST  
**Scope:** JSON student bulk/create-conflict orchestration convergence

## 1. JSON Bulk Create Convergence

- `StudentViewSet.bulk_create` is now a transport facade over `import_students_from_rows()`.
- JSON bulk create now shares the same student import row policy as Excel import:
  tenant-scoped name+parent duplicate detection, deleted-student restore, school-level validation, phone/ps_number/omr derivation, and parent password notice mapping.
- The response keeps the existing `created`, `failed`, and `total` fields and now also exposes the shared `duplicates`, `restored`, and `processed_by` fields.
- Active duplicates no longer create an extra account graph when the student phone differs but name+parent matches.

## 2. Conflict Resolution Convergence

- Added `resolve_student_import_conflicts()` to `apps.domains.students.services.import_students`.
- `StudentViewSet.bulk_resolve_conflicts` now delegates restore and delete-and-recreate row policy to the import service.
- Delete-and-recreate uses the same normalized row creation path as Excel import.
- If delete-and-recreate validation fails, the originally deleted student remains preserved; the permanent delete rolls back.
- Restore still does not send new-password welcome messages.
