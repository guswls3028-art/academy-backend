# V1.2.31 Release Notes

**Status:** production-deployed
**Release date:** 2026-06-05 KST
**Scope:** OMR legacy auto-grading score-shape unification + production rollback proof

This release turns the v1.2.30 decorative-essay fix from a visible-result
repair into a full grading-path unification. v1.2.30 corrected the canonical
`Result` / `ResultItem` / admin API score shape. A follow-up audit found that
the legacy `ExamResult` auto-grading container still had its own equal-split
score calculation. The final visible score was already corrected by the later
sync step, but leaving that duplicate calculation alive would be a structural
risk for future paths that still inspect `ExamResult`.

## 1. Root Cause

- `apps.support.omr.score_shape.get_exam_score_shape()` is now the canonical
  source for:
  - question kind;
  - objective max;
  - subjective max;
  - per-question max;
  - decorative essay detection.
- `sync_result_from_exam_submission()`, session score metadata, result detail
  metadata, and manual item scoring already consumed that canonical shape.
- `ExamGradingService._compute_score()` still used its own legacy rule:
  - if all raw question scores were zero, divide exam max by every sheet
    question;
  - include essay slots in the breakdown/max.
- On a decorative `1 choice + 1 essay` equivalent, that could make the legacy
  `ExamResult` stage calculate `50` instead of `100`, even though the final
  `Result` sync later corrected the user-visible score.

## 2. Fix Summary

- `ExamGradingService` now imports and uses `get_exam_score_shape()`.
- Automatic objective grading now filters out score-shape `essay` questions
  before scoring.
- Per-question max for auto-grading now comes from
  `ExamScoreShape.question_max_score()`.
- Manual override fallback max now uses
  `ExamScoreShape.question_potential_max_score()`, matching the explicit
  manual-item activation behavior.
- Added regression tests for both:
  - decorative zero-score mixed sheet: objective component gets the full max;
  - real structured mixed sheet: objective auto-grading scores only the
    objective component.

## 4. Structural Verdict

This is no longer just a display or result-sync patch. The grading path now has
one canonical interpretation for decorative essay slots:

- visible sheet slots can exist without being score-bearing;
- score-bearing essay slots are activated by explicit evidence such as raw
  score, essay answer key, manual essay item, subjective fact, or positive
  subjective component;
- automatic objective grading, result sync, session metadata, result detail
  metadata, and manual item scoring all consume the same OMR score-shape model.

The remaining long-term cleanup is conceptual rather than urgent: `ExamResult`
still exists as a compatibility container, while `Result` / `ResultItem` /
`ResultFact` are the canonical scoring records. v1.2.31 removes the known
duplicate score calculation risk inside that compatibility path.
