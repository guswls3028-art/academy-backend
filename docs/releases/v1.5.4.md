# V1.5.4 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-06-28 KST
**Scope:** Clean-PDF matchup reading-flow segmentation seal

This patch replaces the clean-PDF question segmentation path used by matchup
analysis with a reading-flow based splitter and closes the follow-up production
gap where legitimate short prompts could be filtered out after segmentation.

## 1. Decision

GO for production use.

Tenant 2 document `303` was reanalyzed on production after deployment and now
returns 83 extracted questions with no missing question numbers. The regenerated
problem crops were downloaded from production storage, reviewed in visual audit
sheets, and checked again in the real admin matchup page through Chrome.

## 2. Change Summary

- Added `clean_pdf_v2`, a reading-flow splitter for embedded-text PDFs that
  uses semantic question anchors, shared source blocks, section headers, and
  layout-aware visual extents instead of dense page slicing.
- Routed clean embedded-text PDFs through the new splitter before the legacy
  dense-fill fallback.
- Preserved segmentation flags through the matchup pipeline so downstream
  filtering can distinguish deliberate short workbook prompts from broken
  fragments.
- Skipped area-based rejection for `clean_pdf_v2` crops, fixing the production
  Q13/Q74-style short-prompt loss.
- Added focused regression coverage for body/source-prefixed anchors,
  dual-column flow boundaries, answer-page rejection, full-width visual
  questions, right-side visual context, and short-prompt preservation.

## 5. Residual Risk

- Automatic segmentation remains best-effort for arbitrary scans, rotated
  photos, or unusually designed worksheets. Manual direct-crop and merge tools
  remain the recovery path for ambiguous layouts.
- The new splitter is intentionally scoped to clean embedded-text PDFs. Scanned
  single-image materials still use the existing OCR/layout paths.
- Further confidence should come from accumulating a golden set of additional
  school PDFs, but the reported document `303` failure class is closed in
  production and visually verified.
