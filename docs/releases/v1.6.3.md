# V1.6.3 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-07-08 KST
**Scope:** ymath/limglish billing contract price exception

This patch release applies the 150,000 KRW monthly contract price for
`ymath` and `limglish` across billing policy, existing Program rows, and
future plan-change operations.

## 1. Decision

GO for production use.

Backend `main` is deployed. The production migration is applied, API runtime
image digest matches CI, production canary passed, and direct live DB checks
confirmed both Program prices and open invoices use the contract amount.

## 2. Change Summary

- Added `Program.resolve_monthly_price()` and tenant-code contract overrides
  for `limglish` and `ymath` at 150,000 KRW.
- Updated billing plan changes to apply the contract price for those tenants
  instead of overwriting them with the standard plan price.
- Updated `audit_billing_fields` so ymath/limglish contract pricing is treated
  as expected policy, not an anomalous manual override.
- Added migration `core.0041_set_ymath_limglish_contract_price` to set existing
  ymath/limglish Program rows to 150,000 KRW.
- Documented the billing contract exception in the billing go-live runbook.

  billing plan changes call it; invoice creation already consumes
  `program.monthly_price`.

  no tenant fallback or cross-tenant query path was introduced.
- Frontend integration audit: no frontend API contract changed; existing billing
