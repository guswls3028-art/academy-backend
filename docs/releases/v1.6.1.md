# V1.6.1 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-07-06 KST
**Scope:** Whole-project refactor closure + stale AI recovery + post-deploy operations/cost seal

This patch release closes the broad cleanup/refactor work after V1.6.0. The
main objective was not to remove product capability, but to reduce legacy
coupling, duplicate support code, stale operational residue, and unclassified
infrastructure cost/waste state before the next project phase.

## 1. Decision

GO for production use.

Backend and frontend `main` are deployed, production canary is green, deploy
verification reports GO, student video playback chain passes from a real
student path, and AWS cost/waste audit found no immediate safe downsize or
deletion target remaining after Batch job definition cleanup.

## 2. Change Summary

- Isolated backend cross-domain runtime dependencies through support modules,
  ports, adapters, read models, and shared contract helpers instead of direct
  legacy model/service coupling.
- Centralized frontend query keys, shared formatters, class combiners, status
  checks, and badge/design-system usage to reduce duplicated view code while
  preserving behavior.
- Added stale AI job reconciliation for terminal matchup documents, including
  safe DONE recovery and AIResult backfill when the source document already
  completed.
- Reconciled one production AI DLQ residue item after verifying the source
  document, job identity, tenant, problem count, and missing result state.
- Added a read-only cost/waste audit script for Budget, Cost Explorer, ECR,
  Batch job definitions, ASG capacity, SQS/DLQ, EC2, EIP, SG, EBS, RDS, and
  Redis posture.
- Deregistered 8 old ACTIVE AWS Batch job definition revisions, then verified
  the retained set is `keep=42, drop=0`.
- Refreshed production reports after deploy, canary, cost audit, resource
  inventory, cleanup dry-run, runtime image consistency, and front connection
  checks.
