# V1.9.2 Release Notes

**Status:** Production-deployed / GO

**Release date:** 2026-07-20 KST

**Scope:** Teacher video-thumbnail rendering incident closure, shared fallback
behavior, responsive regression coverage, and production proof

## 1. Decision

GO. Teacher session video cards and the teacher-wide video list now render the
thumbnail already supplied by the media API. The fix is live on `tchul.com`,
and both the reported session surface and the sibling video-management surface
passed mobile and desktop production verification.

## 2. Incident and Root Cause

- The reported production path was teacher lecture `215`, session `245`, video
  tab. Videos `448` and `449` were `READY`, and the production API returned a
  signed `thumbnail_url` for each.
- R2 contained both JPEG objects at the expected tenant-scoped keys. Object
  probes returned non-zero sizes of `103754` and `100434` bytes, and CDN HEAD
  probes returned HTTP 200 with `image/jpeg`.
- The backend serializer, database rows, storage objects, and current video
  worker publication path were healthy. Historical job logs had aged out under
  the configured 30-day retention, while both stored transcode-job records were
  `SUCCEEDED` with one attempt and no recorded error.
- The defect was frontend-only. `SessionDetailPage.tsx` unconditionally rendered
  a play icon instead of `thumbnail_url`. `VideoListPage.tsx` rendered a derived
  thumbnail only for YouTube items and ignored uploaded-video `thumbnail_url`.

## 3. Fix and Recurrence Prevention

- Added one shared `CompactVideoThumbnail` component for uploaded and YouTube
  videos. It renders the supplied image with a 16:9 cover crop, resets failure
  state when the URL changes, and falls back to the existing play icon for a
  missing URL or image load error.
- Applied the component to both the session video tab and the teacher-wide video
  list so the same API contract cannot diverge across the two surfaces.
- Kept thumbnails decorative (`aria-hidden`, empty image alternative text) to
  avoid duplicating the adjacent video title in button accessible names.
- Added a focused Playwright regression that follows the teacher click path from
  dashboard to lecture, session, and video tab, then opens the global video
  list. It covers 390x844 and 1366x900 layouts, loaded pixel dimensions, a null
  thumbnail, and an HTTP 404 thumbnail fallback.
