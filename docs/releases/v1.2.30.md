# V1.2.30 Release Notes

**Status:** production-deployed
**Release date:** 2026-06-04 KST
**Scope:** OMR decorative essay score-shape hardening + Tenant 2 production regrade

This release closes the Tenant 2 OMR score-shape incident where a teacher built
a 20-question objective exam but kept 5 essay boxes on the printed OMR sheet as
visual decoration. The previous score-shape fallback treated the zero-score
`20 choice + 5 essay` custom sheet as a real 80-point objective + 20-point
subjective exam, which made otherwise correctly recognized OMR submissions look
too low.

## 1. Root Cause

- Tenant 2 exam `361` has:
  - `choice_count=20`
  - `essay_count=5`
  - `max_score=100`
  - raw `ExamQuestion.score=0` for all 25 custom questions.
- There was no essay scoring evidence for the 5 essay slots:
  - no essay answer-key entries,
  - no essay `ResultItem`,
  - no subjective `ResultFact`,
  - no existing positive subjective component.
- The all-zero score fallback still split total score across all 25 slots, so
  each objective question was scored as 4 points instead of 5.
- Recognition itself was not the current failure: the latest Tenant 2 OMR
  submissions had completed recognition runs and 20 objective answers persisted.

## 2. Fix Summary

- `ExamScoreShape.question_max_score()` now resolves per-question max from the
  active objective/subjective component maxes instead of blindly dividing total
  score by all sheet slots.
- Zero-score mixed sheets now distinguish real essay scoring from decorative
  essay boxes:
  - if essay scoring evidence exists, preserve the mixed objective/subjective
    split;
  - if no essay evidence exists, treat essay slots as decorative and assign the
    full exam max to objective questions.
- Added `question_potential_max_score()` for explicit manual item scoring so a
  teacher can still activate a zero-score essay question by entering a manual
  item score.
- `AdminExamItemScoreView` uses the potential max only when creating a missing
  item, while normal API metadata continues to expose the active score shape.

## 4. Audit Notes

- Tenant isolation: the fix only reads scoring evidence for the same exam id or
  its resolved template answer key. The production regrade was limited to
  Tenant 2 submissions `301`, `302`, and `303`.
- Data integrity: existing real subjective scoring evidence keeps the mixed
  objective/subjective split; decorative essay handling applies only when
  essay evidence is absent.
- API contract: session score metadata and result detail metadata now expose
  decorative essay boxes as visible but non-score-bearing slots.
- Frontend: no frontend code change was required for this release because the
  existing v1.2.29 frontend consumes the backend's canonical score-shape
  metadata.

## 5. Operational Note

For this incident, "OMR is still not working" meant the final displayed score
was wrong, not that recognition failed. Recognition was already completing; the
remaining production bug was that decorative essay boxes incorrectly consumed
20 points of the exam total.
