## 2. Change Summary

Backend:

- Converted `ScoreEditDraft.session_id`, `tenant_id`, and `editor_user_id` to
  Django ForeignKey fields while preserving deployed DB column names and the
  public `_id` attnames used by callers.
- Converted `WrongNotePDF.enrollment_id`, `lecture_id`, and `exam_id` to
  ForeignKey fields. Invalid required enrollment refs are deleted during the
  migration; invalid optional lecture/exam refs are nulled to preserve job
  history.
- Converted `HomeworkScore.updated_by_user_id` to nullable `core.User`
  ForeignKey and kept the serializer `updated_by_user_id` response contract.
- Removed the direct `apps.domains.lectures.models.Session` import from
  `HomeworkScore` by using a string ForeignKey reference.
- Updated `check_id_domain_safety.py` to recognize model-level uniqueness for
  `AIJobModel.job_id`, `MatchupDocument.inventory_file`, and
  `SessionProgress(enrollment, session)`, eliminating the remaining
  false-positive `UNORDERED_FIRST` warnings.
- Updated refactor inventory, guardrail docs, integer-FK allowlist, and the
  2026-06-23 whole-project audit report.
