# V1.2.46 Release Notes

**Status:** Production-deployed and tenant 2 rollback-probed
**Release date:** 2026-06-09 KST
**Scope:** OMR decorative essay fail-closed behavior and scored direct-input essay handling

This release sharpens the distinction between two visually similar OMR cases:

- **Scored direct-input essay:** the teacher configured a positive essay point
  value. The answer may be typed later by the teacher or omitted from the answer
  key; the essay is still a real scoring target.
- **Decorative essay space:** the sheet has an essay-looking area but no
  positive essay point value. It is not a scoring target and must not accept
  direct positive score input.

## 1. Decision

OMR mixed objective/essay scoring remains **GO**.

The source of truth for direct-input essay scoring is the configured question
point value, not the presence of `해설참조` or a text answer placeholder.

## 2. Change Summary

Backend:

- New essay `ResultItem` creation now uses `score_shape.question_max_score()`
  instead of `question_potential_max_score()`.
- A zero-score decorative essay row can no longer be promoted into a scored row
  by sending a positive item score to the manual item-score endpoint.
- Existing scored essay rows still accept direct manual scoring when the
  configured essay question score is positive.
- Regression tests now cover:
  - positive essay score + no answer-key entry -> direct subjective score save
    succeeds,
  - decorative essay space -> direct subjective score save is rejected,
  - decorative essay space -> per-item positive score save is rejected,
  - detail/session score metadata continues to expose decorative essay max 0.
