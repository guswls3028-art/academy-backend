# V1.6.0 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-06-29 KST
**Scope:** Idle Batch cost cleanup + runtime env netprobe closure

This infrastructure release closes the local/deploy cleanup pass for the V1
production stack. It aligns the video Batch compute environment with SSOT,
reduces idle ops Batch wakeups, updates runtime evidence, and fixes the worker
runtime API URL sync path that made Batch netprobe fail API connectivity while
public production health was already green.

## 1. Decision

GO for production use.

Production now reports video Batch CE `SPOT`, ops CE `EC2`, both queues enabled,
messaging warm baseline `1`, RDS `db.t4g.medium`, and no structural drift.
The final deploy run completed with `DEPLOY_SUCCESS`; the final netprobe job
`9b938723-fa94-41cd-b065-da082554e145` succeeded.

## 2. Change Summary

- Reduced idle Batch ops cost by moving `academy-v1-enqueue-uploaded-videos`
  from a 10-minute cadence to `rate(1 hour)`.
- Added Batch CE spec drift visibility for type/allocation/max vCPU/instance
  types, plus inventory columns for the same fields.
- Made video CE creation/update honor SSOT `useSpot`, with active-job guards
  before future compute-resource changes.
- Aligned docs with the live RDS `db.t4g.medium` and messaging worker warm
  baseline.
- Synced runtime `API_BASE_URL` into API/workers SSM env from the public API
  domain first, falling back to the ALB URL only when no public domain exists.
- Refreshed production reports after deploy/canary/deploy-verification.
