# V1.2.26 Release Notes

**Status:** production-deployed
**Release date:** 2026-06-02 KST
**Scope:** Student video progress enrollment resolution + limglish production evidence

This release closes the reported limglish teacher incident where students said
they watched assigned videos but the viewing/progress record did not remain
visible to the teacher.

## 1. Root Cause

- The clarified symptom was video watch/progress history, not student course
  application or teacher-created enrollment.
- Student video playback already resolved the correct active enrollment for the
  video's lecture.
- Progress save did not carry that enrollment from the frontend and the backend
  fallback could use the student's first active enrollment instead.
- For students with multiple active enrollments, playback could succeed while
  progress persistence failed or targeted the wrong lecture/enrollment context.
- Production evidence in limglish matched the failure mode: 임근혁 선생님's
  active lectures had video `view_count` activity but sparse or missing
  `VideoProgress` rows, and most enrolled students' first active enrollment was
  a different lecture.

## 2. Fix Summary

- Backend progress save now resolves the active enrollment by the video's
  lecture when no explicit enrollment is provided.
- Backend rejects an explicit enrollment that does not belong to the video's
  lecture.
- Student video list/playback payloads expose `enrollment_id` and
  `last_position`.
- Frontend video progress POST includes the resolved `enrollment_id`.
- Frontend next/list navigation keeps the effective enrollment context.

## 4. Operational Note

The teacher's wording "수강 기록" could have meant course enrollment,
attendance, or video completion. Follow-up clarified it was video watch history.
The self-registration/application flow was inspected and is unrelated: it does
not create lecture/session enrollment or video progress records.

Existing historical missing progress rows are not backfilled because `view_count`
does not identify which student watched the video. New student viewing activity
now persists against the correct video lecture enrollment.
