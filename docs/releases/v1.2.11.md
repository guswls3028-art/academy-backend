# V1.2.11 Release Notes

**Status:** production-deployed  
**Release date:** 2026-05-24 KST  
**Scope:** HJ3 OMR production grading hardening + student launch surface polish

## 1. OMR Recognition And Grading

- Added a shared filled-anchor detector for small black identifier/answer anchors and routed both OMR engine and identifier detection through it.
- Hardened HJ3 scanned-sheet alignment by allowing realistic shifted answer columns while still rejecting implausible anchor matches.
- Added legacy v14 marker fallback support so submitted HJ3 sheets using the older corner-marker visual system can still be read through the current worker path.
- Made incomplete identifiers explicit: raw values such as `11139??2` now become `IDENTIFIER_INCOMPLETE` / `needs_identification` instead of silently looking like a reader miss.
- Kept ambiguous-but-matchable identifiers as safe manual-review cases when another enrolled candidate can plausibly conflict.

## 2. Student Launch Surface Polish

- Polished student dashboard, video, sessions, grades, community, inventory, profile, and rich editor surfaces for the launch pass.
- Replaced the student grades trend dependency on a runtime chart component with a compact SVG chart tailored to the student app surface.
- Allowed data URL video thumbnails so mocked/inline student visual QA and safe fallback previews render consistently.
- Added `scripts/audit-student-rich-content.mjs` to exercise the mobile student app with rich mocked data and capture the launch-critical screens.
- Strengthened structure inventory scripts:
  - `verify-student-routes.mjs` now reads StudentRouter imports directly instead of relying on a hard-coded route list.
  - `refactor-boundary-snapshot.mjs` now reports domain-boundary hotspots and UI/API structure metrics.
