# V1.4.12 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-06-24 KST
**Scope:** Problem Studio bounded OCR + HWPX companion review workbook + remaining-risk closure

This patch closes the largest remaining Problem Studio transfer risks from
v1.4.11. The default workflow is still conservative: teachers upload official
source material, download a review package, and edit/approve the copied content.
The release adds bounded automatic OCR and a text-focused HWPX companion
workbook without promoting beta rewrite to the primary workflow.

## 2. Change Summary

Backend:

- Added `apps.domains.tools.problem_studio.ocr` for bounded local Tesseract OCR
  on raster image/PDF units. OCR failures, missing binaries, missing language
  packs, empty results, and per-package limit exhaustion degrade to the existing
  OCR work queue instead of raising.
- Added `PROBLEM_STUDIO_OCR_MAX_UNITS` with default 8 units per transfer
  package and `PROBLEM_STUDIO_OCR_ENABLED` kill switch.
- Added `apps.domains.tools.problem_studio.hwpx_writer` to produce
  `03_자체양식_문제검수본.hwpx`, a text-focused HWPX companion workbook.
- Updated the transfer manifest and review workbook with OCR completed/pending
  unit counts, per-document OCR status/engine/warning fields, and
  `native_hwpx_output=true` / `native_hwp_output=false`.
- Updated the API Docker image to install `tesseract-ocr` and
  `tesseract-ocr-kor`, and added `pytesseract` to API requirements.
- Updated fixture tooling, domain docs, and UAT runbook for the 7-file review
  bundle and bounded OCR behavior.

Frontend:

- Updated Problem Studio copy to present HWPX as a companion review workbook.
- Updated success/warning copy to distinguish remaining OCR candidates from
  OCR already handled by the server.
- Kept the primary CTA as the conservative transfer workflow and left beta
  rewrite collapsed.

Operations and cost:

- No external OCR provider or new standing service was introduced. OCR runs
  inside the existing API image and is capped per transfer package.
- GitHub Actions ECR cleanup ran after deploy with the normal manifest-aware
  retention policy.
- API min/desired capacity remains 2. Messaging remains 1. Tools remains 1.
  AI worker remains desired 0 and live refresh is not applicable while scaled
  down.

## 6. Product Review

The product promise is now cleaner for real teacher review:

- Safe default: "원본을 그대로 옮긴 편집 가능한 검수 패키지"
- Stronger conversion point: small scanned/image units can become editable text
  immediately through OCR.
- Clear fallback: over-limit or unreadable scan pages are not silently lost;
  they remain in `02_OCR_연결후보.csv`.
- Better Hangul story: teachers get `.doc` compatibility output plus a native
  HWPX companion workbook for text review.

The remaining business gap is not "can we move source files?" anymore. It is
template fidelity: exact HWP/HWPX layout with images, tables, equations, and
endnotes. That is the next high-leverage paid-upgrade path.

## 7. Residual Risk

- Binary native `.hwp` writing is still not shipped.
- The HWPX output is a text-focused companion workbook, not exact original
  layout reconstruction.
- OCR is bounded and best-effort. Dense Korean scans, formulas, tables, and
  diagrams still need teacher review or future async OCR/template rendering.
- Beta rewrite remains a teacher-reviewed candidate generator, not final
  classroom material.
- `django check --deploy` still reports the existing `SECURE_SSL_REDIRECT`
  warning inside the app container; edge HTTP-to-HTTPS redirect, canary, and
  health checks passed.
