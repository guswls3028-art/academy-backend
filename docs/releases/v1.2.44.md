# V1.2.44 Release Notes

**Status:** Production-deployed and tenant 2 OMR exam 419 regraded
**Release date:** 2026-06-09 KST
**Scope:** OMR custom question weights, mixed objective/essay score shape, and production score repair

This release closes the follow-up OMR scoring incident for tenant `tchul`
(tenant id 2), exam 419. The visible symptoms were:

- OMR recognition could complete, but teacher-entered custom point values were
  ignored.
- Existing results were scored from fallback/equal distribution rather than
  the intended question scores.
- A stale `Result.total_score - objective_score` difference could survive a
  regrade as if it were a real manual subjective score.

## 1. Decision

OMR scoring for custom mixed sheets is **GO** for the covered production path.

The production exam now has the intended score source of truth:

- Questions 1-18: 3 points each.
- Questions 19-22: 4 points each.
- Question 23: 30-point subjective/essay question.
- Score shape: objective 70, subjective 30, total 100, source `sheet`.

## 2. Root Cause

There were two independent reset paths.

1. The frontend shape-apply call always sent `choice_score: 0` and
   `essay_score: 0`. The backend interpreted those fields as an explicit bulk
   score update, so existing `ExamQuestion.score` rows were overwritten to 0.
2. Exam 419 is a regular exam without `template_exam_id`, so it owns its own
   sheet/questions. The frontend only allowed question score patching when
   `exam_type === "template"`, so the follow-up score save was skipped even
   though the backend model permits regular-without-template as the structure
   owner.

During repair, one more scoring durability bug was found: regrading preserved
any existing `Result.total_score - Result.objective_score` delta as subjective
score, even without a `manual_subjective` fact or manual essay item. That could
carry stale totals forward.

## 3. Change Summary

Backend:

- Question shape initialization now supports count-only updates without
  rewriting existing question scores.
- Legacy zero-score payloads no longer clear existing positive scores.
- Regular exams without a template are treated as editable structure owners;
  template-bound regular exams remain locked to their template.
- Mixed objective/essay OMR score shape treats explanation placeholders such as
  `해설참조` as non-scoring unless there is actual scoring evidence.
- Result sync now preserves subjective score only from explicit evidence:
  `manual_subjective` facts, manual essay `ResultItem`s, or manual-subjective
  attempt metadata. Stale total/objective differences are dropped on regrade.

Frontend:

- Shape apply omits score fields unless the user explicitly applies auto-score
  values.
- Auto-score disabled sections preserve existing per-question scores.
- Regular-without-template exams can patch their own question scores.
- Apply success now uses mutation variables rather than stale React state.

Production repair:

- Tenant 2 exam 419 question scores were restored to `3/4/30`.
- Existing OMR submissions 349-353 were regraded with the deployed production
  code.
- No manual subjective facts/items existed for those rows, so stale subjective
  deltas were removed.
