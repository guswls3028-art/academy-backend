# V1.4.2 Release Notes

**Status:** Production-deployed / GO  
**Release date:** 2026-06-21 KST  
**Scope:** Tenant 2 26-1m color workbook segmentation hardening

This patch closes the specific bottleneck found in the `26-1m` style color and
decorated workbook materials. These documents mix embedded text, scan-like
layout, concept-note pages, multi-column problem bodies, and parenthesized
subitems. The goal was to make the real segmentation behavior succeed, with
tests and audits as evidence of that behavior rather than as a shortcut.

## 1. Decision

GO for current production operation.

For the known Tenant 2 `26-1m` workbook set, the splitter now recovers all
manual physical GT problem anchors in the audited scan/PDF materials. Hand-shot
photos remain explicitly out of scope.

## 2. Change Summary

Backend:

- Restored dual-column scan splitting when left/right top-level body anchors are
  present, not only when marginal anchors are detected.
- Prevented dense fill-in-row detection from treating parenthesized body
  subitems like `(1)` as top-level question anchors.
- Kept parenthesized body subitems under their parent problem region instead of
  emitting them as separate question boxes.
- Expanded concept-note page detection for decorated/color chemistry workbook
  pages such as periodic-table, chemical-bonding, ionic-bonding, metallic-bond,
  semiconductor, diode, and resistance notes, while preserving pages with real
  question indicators.
- Added targeted regression coverage for the 26-1m color workbook page where
  Q5/Q6/Q7/Q8 previously collapsed or split incorrectly.

Frontend:

- Not changed.
