# V1.9.3 Release Notes

**Status:** Production-deployed / GO

**Release date:** 2026-07-23 KST

**Scope:** Mixed per-question OMR types, math numeric short-answer grading,
teacher Excel result import, and production real-use closure

## 1. Decision

GO. A teacher can set the total question count, assign objective or short-answer
type to each question in any order, generate the matching mixed OMR sheet, and
import an already graded workbook after previewing student matches, question
outcomes, scores, and overwrite effects. Math numeric answers from 0 through 999
are normalized and auto-graded on direct-answer entry.

Handwritten short-answer OCR is not part of this seal. Mixed sheets provide the
short-answer writing areas, while those results can be entered directly or
imported from the teacher's graded Excel workbook. Physical printer, pen, and
scanner acceptance remains an external hardware check; the production PDF,
digitally marked OMR recognition chain, and scanner-like automated fixtures pass.

## 2. Delivered Behavior

- Exam setup owns an explicit per-question type list and preserves arbitrary
  objective/short-answer order as the question count changes.
- OMR PDF generation renders answer bubbles only for objective questions and
  numbered writing areas for short-answer questions.
- Math numeric answers accept 0-999, reject out-of-contract content, normalize
  leading zeroes, and compare numerically for automatic grading.
- The result import accepts the product template and teacher workbooks containing
  leading title rows, extra columns, name or enrollment identity, phone columns,
  numeric question headings, and blank/O/X correctness marks.
- Import is preview-first. It reports unmatched or ambiguous rows, per-student
  wrong questions and score, and whether an existing result will be overwritten
  before the teacher applies it.
- A downloadable exam-specific workbook is available in the grading/results tab.
- Workbooks whose worksheet XML omits the optional `dimension` element are
  supported; a real generated XLSX exposed this edge case and a regression fixture
  now strips that element deliberately.

## 5. Audit Lenses and Final State

- Functional: every requested setup, OMR layout, numeric answer, Excel preview,
  apply, overwrite, and result-projection path has executable coverage.
- Security and tenancy: imports require the existing exam-result permissions and
  resolve students only inside the active tenant and exam enrollment set.
- Data and schema: the question-type migration is applied; import application is
  transactional and preserves the existing exam-attempt history contract.
- Edge cases: arbitrary mixed order, question-count changes, leading title rows,
  extra summary columns, phone/name matching, malformed marks, ambiguous students,
  and dimensionless workbooks are bounded.
- UX and accessibility: desktop/mobile layouts, button names, status feedback,
  preview table, overwrite warning, horizontal table scrolling, and fixed mobile
  navigation were inspected.
- Operations and documentation: runtime manifests, generated verification reports,
  domain docs, user guides, release index, CI, canaries, and cleanup evidence agree.

No release-scoped Critical or High finding remains. The only explicit boundary is
physical handwriting/scanner hardware acceptance; handwritten answer recognition
is not represented as an automatic-grading capability.
