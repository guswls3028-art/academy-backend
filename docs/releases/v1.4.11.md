## 6. Product Review

The product is now coherent enough for beta customer review because the default
workflow does not overpromise AI rewriting. A teacher can upload EBS/reference
materials, receive a single ZIP, open the checklist first, review the
self-template workbook, and use the OCR CSV for raster-only pages that need
manual transcription or future OCR processing.

The strongest business message is:

- stable default: "원본을 그대로 옮긴 편집 가능한 검수 패키지"
- visible safety: checklist, manifest, file list, review workbook, OCR queue
- optional accelerator: beta rewrite, clearly not final answer material

This lowers support risk because failure cases become review tasks instead of
silent omissions.

Next best investments:

- Async OCR worker that consumes `02_OCR_연결후보.csv` candidates and regenerates
  the review workbook.
- Native `.hwp`/`.hwpx` export fidelity beyond current Hangul-compatible HTML
  `.doc`.
- Page-level preview and selective export for very large publisher packages.
- Teacher correction telemetry to learn which extracted blocks are repeatedly
  edited.

## 7. Residual Risk

- Current editable output is Hangul/Word-compatible `.doc` HTML, not native
  binary `.hwp`.
- Raster-only PDF/image pages transfer visually and are now explicitly queued in
  `02_OCR_연결후보.csv`, but automatic OCR text extraction is not yet shipped.
- Beta rewrite remains a teacher-reviewed candidate generator, not final
  classroom material.
- `django check --deploy` still reports the existing `SECURE_SSL_REDIRECT`
  warning inside the app container; edge HTTP-to-HTTPS redirect, canary, and
  deployment health checks passed.
- AI worker live refresh is not applicable while desired capacity is 0; this is
  the current intended cost posture and is verified by canary.
