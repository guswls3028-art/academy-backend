# V1.7.0 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-07-10 KST
**Scope:** Production cost cleanup: DNFM right-sizing + unused RDS Proxy retirement

This infrastructure release removes fixed monthly cost that no longer serves the
runtime path. The Academy API and workers already connect directly to
`academy-db`, so the unhealthy and unused RDS Proxy was retired. The separate
`dnfm-origin` EC2 instance was right-sized from `t4g.small` to `t4g.micro`.

## 1. Decision

GO for production use.

Academy runtime stayed on direct RDS, API health remained green, ALB target
health stayed healthy, and DNFM returned HTTP/HTTPS 200 after the instance type
change.

## 2. Change Summary

- Changed `dnfm-origin` (`i-0b2834844f14869f4`) from `t4g.small` to
  `t4g.micro`.
- Preserved DNFM Elastic IP `43.202.246.97` and root gp3 volume
  `vol-03740191043c8d81c`.
- Deleted unused RDS Proxy `academy-db-proxy`.
- Scheduled the proxy-only Secrets Manager secret
  `academy/db/proxy-credentials` for 7-day recoverable deletion
  (`DeletionDate=2026-07-17 17:09 KST`).
- Updated infra SSOT to mark RDS Proxy as not configured.
- Updated the ops healthcheck so direct-RDS runtime is treated as normal and
  Auto Scaling policy alarms are informational, not service incidents.
- Updated the disaster recovery runbook so RDS Proxy steps are optional and
  only apply if a proxy is reintroduced.

## 3. Cost Impact

- RDS Proxy removed: July MTD proxy charge was `$7.67`; June full-month proxy
  charge was `$25.92`.
- Proxy-only Secrets Manager secret scheduled for deletion: avoids the ongoing
  secret storage/API request charge after the recovery window.
- DNFM compute right-sized: `t4g.micro` halves the instance size from
  `t4g.small`; Elastic IP and EBS costs intentionally remain because the
  service stays online.

## 6. Audit Lenses

- Runtime DB audit: API and worker SSM values point directly at the RDS instance
  endpoint; no runtime path uses the retired proxy.
- Availability audit: RDS instance, API ASG, messaging worker ASG, SQS queues,
  Redis, Batch queues, and API ALB target stayed healthy after the cleanup.
- Cost audit: RDS Proxy and proxy-only secret were the only no-runtime-use
  billable resources removed immediately. Redis/RDS downsizing remains deferred
  because it requires service restart or capacity risk.
- Rollback audit: DNFM can be returned to `t4g.small`; the secret is recoverable
  until `2026-07-17 17:09 KST`. Reintroducing RDS Proxy would require creating
  a new proxy and updating SSM `DB_HOST`.

## 7. Deferred Items

- Redis `cache.t4g.small` uses about `1.1%` memory and has no evictions, but
  node class changes restart the single-node cache, so no immediate change was
  made.
- RDS `db.t4g.medium` has low CPU, but downsizing would require DB restart and
  memory headroom is not generous enough for a no-risk change.
- Public IPv4 charges remain because the API, messaging worker, ALB, and DNFM
  still need their current network paths.
