# V1.2.43 Release Notes

**Status:** Production-deployed and OMR multi-mark recognition verified
**Release date:** 2026-06-08 KST
**Scope:** OMR clear multi-mark detection, dense custom-sheet layouts, and review UI multi-select display

This release closes the OMR regression reported from a custom sheet whose
question count increased enough to change the dense layout/layer structure. The
visible symptom was a double-marked objective answer being treated as a single
choice in recognition/review.

## 1. Decision

OMR multi-mark recognition is **GO** for the covered objective-sheet paths.

The exact teacher-supplied Kakao PDF/photo was not present in the workspace, so
the direct proof uses synthetic OMR fixtures that exercise the same detector
contract across dense 30/45/60-question layouts, including early-row, later
column, and final-question positions.

## 2. Change Summary

Backend OMR engine:

- Clear multi-mark detection no longer depends only on a small top-two score
  gap.
- After blank/noise-floor filtering, multiple confidently marked bubbles now
  return `marking="multi"` with all detected choices, even when one mark is much
  darker than the other.
- Ambiguous small-gap handling remains in place for weak/noisy marks.
- Added dense-layout regression coverage for unbalanced double marks across
  30/45/60-question sheets.

Frontend OMR review:

- Objective answer rows now parse stored multi-choice answers such as `3,4`
  through the shared choice-answer parser.
- Review bubbles highlight every selected choice instead of falling back to
  "미선택" for comma-separated values.
- Button clicks now toggle membership and preserve the canonical comma-joined
  answer string.

## 5. Residual Risk

- The exact Kakao-attached PDF/photo was not available locally, so this release
  is verified by detector-contract fixtures and the existing real-use OMR
  simulator rather than that precise scan.
- Very faint or noisy second marks may still land in the existing ambiguous
  review path by design.
- Production workflow warnings about missing student-video chain-smoke secrets
  and zero-capacity AI ASG are not blockers for this OMR fix, but remain
  operational cleanup items for broader deploy verification fidelity.
