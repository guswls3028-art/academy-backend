# V1.4.5 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-06-23 KST
**Scope:** Whole-project optimization audit + tenant metadata hardening + mobile submissions visual QA

This patch seals the project-wide optimization audit tranche. It reduces
backend boundary and ID-domain debt, consolidates frontend display/query
contracts, hardens one tenant metadata batch-read edge in results landing
stats, and closes the submissions inbox mobile visual regression found during
production screenshot review.

## 2. Change Summary

Backend:

- Reduced ID-domain safety warnings from 52 to 32 with 0 errors.
- Reduced refactor boundary counts to:
  - `cross_domain_import=117`
  - `cross_domain_internal_import=591`
  - `domain_infra_import=81`
- Added and routed through repository/public-query boundaries for enrollment,
  exams, submissions, homework, and inventory read paths touched by progress,
  matchup, messaging, results, student-result, and pending-submissions flows.
- Added deterministic ordering to the audited `.first()` paths that could be
  fixed without widening strict boundary debt.
- Hardened `AdminResultsLandingStatsView` tenant metadata resolution so
  enrollment/student and target-title maps are tenant-scoped, even when
  contaminated IDs appear in tenant-scoped submission rows.
- Added a regression test proving contaminated cross-tenant metadata IDs do not
  leak `student_name` or `target_title` into pending results.
- Refreshed deployment evidence reports after production rollout.

Frontend:

- Consolidated repeated display fallback helpers into
  `src/shared/utils/displayText.ts`.
- Added shared assessment query keys and assessment homework-policy/list
  contracts so sessions no longer import admin homework internals for those
  reads.
- Reduced `same_app_domain_import` from 150 to 146 and removed the
  `app_admin/sessions -> app_admin/homework` pair from the top-10 hot-pair
  list.
- Wrapped submissions inbox row actions on mobile so `학생 지정` / `폐기`
  controls stay visible without horizontal overflow.

Docs:

- Updated refactor inventory, phase-0 guardrails, roadmap, frontend snapshot,
  and the 2026-06-23 whole-project audit report.
- Updated current deployment evidence reports and kept the generated history
  snapshots for this verification pass.
