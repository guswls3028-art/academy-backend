# V1.4.1 Release Notes

**Status:** Production-deployed / GO  
**Release date:** 2026-06-21 KST  
**Scope:** Tenant 2 matchup scan/PDF question segmentation recovery

This release closes the Tenant 2 matchup first-gate failure mode: question
segmentation for scanned/PDF source documents. The target was not a narrow test
patch, but durable recovery across the historic Tenant 2 scanned/PDF corpus,
excluding hand-photographed images.

## 1. Decision

GO for current production operation.

The matcher segmentation path now handles sparse text-layer scans, partial text
overlays, workbook-style scan layouts, dense fill-in rows, page restarts, and
non-question cover/decorative pages without preserving the old fragile behavior
as the canonical path.

## 2. Change Summary

Backend:

- Added PyMuPDF word extraction support so PDF pages can distinguish real text
  documents from scanned pages with sparse or partial OCR overlays.
- Reworked matchup PDF dispatch to route sparse/partial overlay scans through
  the scan-layout detector instead of misclassifying them as text sheets.
- Added scan anchor numbering, scan page backfill, terminal cover-page dropping,
  and aggressive scan fragment merging for workbook/scanned layouts.
- Hardened question splitting for wrapped section labels, dense fill-in rows,
  page number restarts, real `zb` choice markers, and isolated OCR garbage
  numbers.
- Expanded golden-eval and manifest-audit commands so known manifest drift is
  reported as data quality evidence instead of hidden test failure.

Frontend:

- Not changed.
