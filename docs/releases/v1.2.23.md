# V1.2.23 Release Notes

**Status:** production-deployed
**Release date:** 2026-06-02 KST
**Scope:** OMR structured essay scoring + manual score UX + session summary score semantics

This release closes the OMR/manual-score loop for mixed objective + essay exams.
Objective OMR answers are auto-graded, teacher-entered essay scores are bounded
by the exam shape, and the final total is now verified as:

`objective_score + subjective_score = score`

Objective-only sheets may still render a decorative essay-looking area for
visual balance, but that area has no grading/input effect.

## 1. Root Cause Themes

- The system previously treated "visual essay area" and "grading essay
  questions" too loosely across OMR, result payloads, and teacher score entry.
- Manual score assignment needed server-side shape enforcement, not just UI
  guidance.
- Session summary exposed `max_score` as the observed highest score in one path,
  which made a 95/100 canary look like "만점 95점" in the UI.
- Objective-only sheets needed a friendly printed layout while remaining
  logically objective-only for recognition, scoring, and manual entry.

## 2. Fix Summary

- Backend score shape now derives canonical grading bounds:
  - `choice_count`
  - `essay_count`
  - `objective_max_score`
  - `subjective_max_score`
  - `score_shape_source`
- Manual score APIs reject:
  - essay scores above the allowed essay maximum.
  - positive essay scores on objective-only sheets.
- Session score payloads expose shape metadata so frontend controls can render
  the correct editable cells.
- Result detail items expose question number and question kind, so answer
  breakdowns split objective/essay by exam shape instead of answer heuristics.
- Frontend score entry shows essay input only when the exam has scored essay
  questions, validates the range, and treats objective-only decorative essay
  areas as non-input.
- Session exam summary now separates:
  - `max_score`: configured exam maximum.
  - `highest_score`: highest observed result score.

  objective auto-score plus teacher-entered essay score.
- Objective-only audit: pass. Decorative essay space remains print/display only.
- UI/UX audit: pass. Teachers see an essay input only when there is a scored
  essay component; objective-only exams do not present dead input controls.
- Hidden-bug audit: pass. The summary `max_score` semantic bug was found and
  fixed before final signoff.
- Tenant isolation audit: pass. Tenant 2 canary data was scoped and fully
  removed after verification.
- Legacy cleanup audit: pass for changed surfaces. Score-shape logic now lives
  behind the OMR support boundary, and touched frontend/backend score paths no
  longer carry the old visual-vs-grading ambiguity.

## 5. Notes

- The low score concern was not reproduced as a scoring-lowered-by-essay bug in
  the final canary. A controlled mixed exam scored 95/100 from 80 objective +
  15 essay, and an objective-only decorative sheet scored 100/100 with no essay
  score contribution.
- Existing real student low scores should still be interpreted from their saved
  objective answer facts and answer keys. The verified scoring pipeline no
  longer reads decorative/blank essay space as scored questions.
