# V1.2.9 Release Notes

**Status:** production-deployed  
**Release date:** 2026-05-23 KST  
**Scope:** Clinic participant transition SSOT + clinic operations mobile QA

## 1. Clinic Participant Transition Convergence

- Added `apps.domains.clinic.services.lifecycle` as the canonical write boundary for participant status transitions, completion, and completion cancellation.
- Added `STAFF_STATUS_TRANSITIONS`, `STUDENT_STATUS_TRANSITIONS`, and `COMPLETE_ALLOWED_TRANSITIONS` so transition policy is no longer embedded in the HTTP view.
- `ParticipantViewSet.set_status`, `complete`, and `uncomplete` now act as transport facades over the clinic lifecycle service.
- Notification payload selection is returned as `ClinicNotificationEvent`; the view remains responsible only for `transaction.on_commit()` dispatch through the existing AlimTalk compatibility path.
- `cancelled` and `rejected` are terminal for staff status changes. Student-side status changes remain limited to cancelling the student's own pending booking.
- `complete` is a distinct use-case: pending/booked participants become attended and receive `completed_at`; cancelled/rejected participants cannot be completed.

Remaining compatibility boundary:

- Participant creation and student booking change orchestration still live in `apps.domains.clinic.views.participant_views`.
- `change_booking` keeps its guarded cancel-after-new-booking flow; extraction to `clinic.services.change_booking` is the next clinic slice.

## 2. Production PostgreSQL Lock Fix

- Production QA exposed PostgreSQL rejecting `SELECT FOR UPDATE` on a nullable outer join when `SessionParticipant.session` was joined in the lock query.
- `_locked_participant()` now locks only the participant row and active student relation; nullable `session` is loaded after the locked query.
- Added a regression assertion so the lock query does not join `clinic_session`.

## 3. Clinic Operations Mobile Layout

- Frontend commit `99802475` added an optional `DomainLayout.className` escape hatch for domain-specific responsive fixes.
- Clinic tabs were compacted on mobile (`진행중`, `메시지`) and scoped with `clinic-domain-layout`.
- Clinic operations header actions, confirmation bar, and participant-card action rows now wrap within the mobile viewport instead of clipping horizontally.
