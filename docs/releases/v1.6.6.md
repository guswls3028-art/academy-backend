# V1.6.6 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-07-09 KST
**Scope:** Ymath anonymous billboard mode + enterprise score analytics + admin tree/mobile visual polish

This patch release closes the five-session work bundle as one production seal.
It enables Ymath's anonymous score billboard operating mode, adds tenant-scoped
score analytics APIs and dashboards, and polishes the admin tree/settings
surfaces that were touched during the rollout.

deployment verification is GO, frontend Cloudflare Pages deployment is green,
and Ymath real-use API plus visual QA passed after the final frontend deploy.

## 2. Change Summary

- Enabled Ymath feature flags for `section_mode=true`, `clinic_mode=regular`,
  and `score_output_mode=anonymous_billboard`.
- Added migration `core.0042_set_ymath_regular_clinic_billboard` and repaired
  `setup_three_tenants` so new/repaired Ymath programs keep the same operating
  mode.
- Added tenant-scoped score analytics BFF logic under
  `apps/support/results/enterprise_analytics.py`.
- Added staff API `GET /api/v1/results/admin/analytics/`.
- Added student/parent API `GET /api/v1/student/grades/analytics/`.
- Added teacher "운영 분석" and student "추이 분석" dashboard panels.
- Moved the analytics BFF into the support boundary after the strict refactor
  guard caught the initial domain-boundary placement.
- Unified admin tree toolbars and expand/collapse controls across the touched
  tree navigation surfaces.
- Fixed and polished Ymath mobile settings mode badges so long Korean labels
  wrap without clipping or syllable-level breakage.
