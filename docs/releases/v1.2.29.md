# V1.2.29 Release Notes

**Status:** production-deployed
**Release date:** 2026-06-03 KST
**Scope:** OMR score-shape question max SSOT + manual scoring UI contract

This release closes a remaining OMR edge after v1.2.28: grading and aggregate
score calculation already used the structured OMR score shape, but some
teacher-facing scoring metadata could still read raw `ExamQuestion.score`
directly. On custom sheets where raw question scores are stored as `0`, that
could make per-item manual score inputs appear capped at `0` even though the
exam-level score shape correctly resolves the question max.

## 1. Root Cause

- Tenant 2 exam `333` is a mixed custom OMR sheet:
  - `choice_count=20`
  - `essay_count=5`
  - `max_score=100`
  - raw `ExamQuestion.score=0` for the custom questions.
- v1.2.28 fixed OMR recognition/result materialization and aggregate manual
  essay composition.
- Remaining drift:
  - `SessionScoresView` exposed `meta.exams[].questions[].max_score` from raw
    `ExamQuestion.score`.
  - `AdminExamResultDetailView` exposed only existing `ResultItem` rows, so the
    frontend filled missing questions from `/exams/{id}/questions/`, which also
    used raw question scores.
  - `StudentResultDrawer` therefore had a path where missing essay questions on
    a zero-score custom sheet could render as `/0` and clamp input to `0`.

## 2. Fix Summary

- Added `ExamScoreShape.question_max_score()` as the canonical per-question max
  resolver.
- Replaced the duplicated private item-score fallback with the score-shape
  method.
- `SessionScoresView` now emits question `max_score` from the score-shape SSOT.
- `AdminExamResultDetailView` now emits a full `questions` payload:
  - `question_id`
  - `number`
  - `kind`
  - score-shape-resolved `max_score`
- `StudentResultDrawer` now prefers the admin detail API's canonical questions
  and uses `/exams/{id}/questions/` only as a legacy fallback.

## 4. Operational Note

The remaining issue was not OMR recognition. It was an API/UI contract drift:
some screens could still ask raw question rows for scoring limits instead of
the OMR score-shape SSOT. The ideal target structure is now in place for this
surface: scoring, session summary metadata, result detail metadata, and the
teacher manual-entry UI all consume the same score-shape-resolved max scores.
