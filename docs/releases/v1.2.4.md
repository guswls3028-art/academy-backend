# V1.2.4 Release Notes

**Status:** production-deployed; final seal pending optional device-body content validation  
**Release date:** 2026-05-23 KST  
**Scope:** student creation account graph SSOT, Excel welcome toggle contract, teacher mobile create UX

## 1. Student Creation Account Graph

- Student account creation is centralized in `apps.domains.students.services.creation.create_student_account`.
- Direct student create, Excel worker create, registration approval, and lecture enrollment new-student create now share the same Parent ensure -> User password/hash -> Student -> active TenantMembership path.
- The service accepts either a plain initial password or a pre-hashed registration password, so registration approval no longer owns a separate account creation graph.
- Parent initial password notice fields remain centralized through the creation result, including phone-keyed parent password reuse for bulk flows.

## 2. Excel And Welcome Message Contract

- Backend Excel upload and worker payloads now parse `send_welcome_message` as an explicit boolean, so string `"false"` no longer becomes truthy.
- Admin Excel upload and teacher mobile Excel upload pass the welcome toggle through the shared frontend contract.
- Teacher mobile Excel upload registers the async worker job in the shared work-status store and tells the operator to check the job box.

## 3. Frontend Contract And UX

- Teacher mobile student creation now uses the shared `createStudent` API contract instead of a teacher-only duplicate payload path.
- Teacher mobile create intentionally sets `sendWelcomeMessage: false`, matching the current no-surprise mobile create behavior.
- The teacher shared `BottomSheet` now constrains its internal scroll area with `overflow: hidden` on the sheet and `min-height: 0` on content, preventing long forms from extending outside the mobile viewport.
- Student create action remains visible in the mobile viewport after the sheet slide-up animation completes.

## 4. Documentation

- Added `docs/domain/student-creation.md` as the student creation SSOT.
- Updated `docs/domain/README.md` and `docs/domain/parent-account.md` to point account creation flows at the new service.
- Updated structure-reform audit docs to mark the student creation account graph duplicate as completed.
