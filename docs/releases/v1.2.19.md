# V1.2.19 Release Notes

**Status:** production-deployed
**Release date:** 2026-06-02 KST
**Scope:** OMR custom sheet shape matrix hardening

This release closes the custom-sheet verification gap after the mixed
objective/essay OMR incident. The OMR shape contract now covers objective-only,
mixed objective+essay, essay-only, large objective, and legacy answer-key
inference paths with the same resolver used by payload dispatch and OMR
document generation.

## 1. Root Cause

- v1.2.17 fixed the reported `20 objective + 5 essay` incident, but the custom
  sheet surface still had unverified edge types.
- Legacy sheet inference recognized only comma-separated `1`-`5` answers as
  objective. It could misclassify valid objective variants such as circled
  digits or alternatives (`2|4`) as essay boundaries.
- OMR document defaults treated `choice_count=0` as "fall back to 20 objective
  questions", which was wrong for essay-only custom sheets.
- The OMR dispatch payload only attached `question_count` and `template_meta`
  when `choice_count > 0`, leaving essay-only sheets without an explicit
  zero-objective contract.

## 2. Fix

- Reused the canonical answer-set parser for legacy OMR shape inference:
  - circled digits, required multi-mark answers, and alternatives are now all
    recognized as objective when every token is in `1`-`5`.
  - non-choice text still marks the start of the essay segment.
- OMR document defaults now preserve a real `mc_count=0` when the sheet has
  questions and is essay-only.
- OMR dispatch payloads now include explicit shape metadata even when
  `choice_count=0`:
  - `question_count=0`
  - `mc_count=0`
  - `essay_count=N`
  - `template_meta.layout.n_cols=0`
- The legacy `generate-omr` endpoint no longer turns `mc_count=0, essay_count>0`
  into a default objective sheet.

## 3. Test Matrix

Added/locked tests for:

- Objective-only custom sheet: `12 + 0`.
- Mixed objective/essay custom sheet: `20 + 5`.
- Essay-only custom sheet: `0 + 3/5`.
- Large objective custom sheet: `45 + 0` with 3 OMR columns.
- Legacy answer-key inference with variants:
  - `①`
  - `2,3`
  - `2|4`
  - then essay text.
- Legacy answer-key inference for essay-only sheets.
- Existing template-copy coverage still preserves `choice_count` and
  `essay_count`.

## 5. Notes

- This does not auto-grade essay answers. It ensures the OMR sheet, generated
  document defaults, worker payload, and legacy inference all agree on how many
  objective bubbles exist for each custom sheet type.
- The original tenant 2 submitted scores from v1.2.17 remain unchanged; this
  release broadens the guardrail so adjacent custom sheet shapes do not regress.
