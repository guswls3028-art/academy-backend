# V1.2.40 Release Notes

**Status:** E2E-verified and sealed; no runtime deployment required
**Release date:** 2026-06-07 KST
**Scope:** OMR browser upload/review/regrade real-use chain and student-domain launch-readiness SSOT update

This release closes the OMR browser-chain launch gap from v1.2.39. No backend
or frontend runtime code changed in this release; the change is a production
real-use canary plus launch-gate documentation.

## 1. Decision

Controlled/internal expansion remains **GO**.

Broad public promotion or large external expansion remains **NO-GO / HOLD**
until the remaining launch gates are passed or explicitly accepted as documented
exceptions:

- account-recovery activation browser chain;
- failed result -> clinic target -> clinic booking/attendance -> remediation;
- beginner/misuse behavior suite across double-submit, stale tabs, mobile
  keyboard, back/forward, and repeated role switching.

## 2. Change Summary

- Added `frontend/e2e/admin/omr-review-realuse.spec.ts`, a production real-use
  canary for generated OMR PDF upload, worker answer persistence, OMR review
  student selection, manual answer correction, regrade, and student grades
  projection.
- Updated launch-readiness, Phase 2 audit, and frontend real-use E2E inventory
  docs so OMR upload/match/review is no longer listed as an open P0 launch gap.
- Recorded cleanup/residue disposition: active generated OMR lectures, sessions,
  and students are `0`; inactive archived exams `399` and `400` are expected
  result-history rows protected by the delete guard.
