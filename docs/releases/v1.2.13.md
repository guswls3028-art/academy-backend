# V1.2.13 Release Notes

**Status:** production-deployed
**Release date:** 2026-05-30 KST
**Scope:** tchul (박철과학) QnA incident response — show student attachment images inline, send QnA alerts only to the owner role, finish wiring the public alimtalk envelopes, dispatch clinic_cancelled on session destroy

This release is the live response to a Saturday-evening tchul incident:
a student asked a question with an image, the AI matchup_search_qna
worker was warming up, and during that 5–10 minute window the teacher
could not see the attached photo at all. Meanwhile QnA alerts were
fanning out to every staff role (including the assistant who is off
duty after hours). All four fixes ship as one release with full
operator visibility.

## 1. QnA Attachment Image Surfacing

- `PostAttachmentSerializer` now returns `download_url` — a 1-hour R2
  presigned URL built from `r2_key`. Existing
  `/posts/<id>/attachments/<att_id>/download/` endpoint is unchanged;
  this is an additive field on the same payload.
- `QnaInboxPage` renders the student's image attachments directly under
  the post body, above the AI matchup result block. The teacher can
  open the image and answer immediately — the matchup result still
  fills in when the worker finishes, but it is no longer a prerequisite
  for seeing the photo.
- The shared `PostAttachment` type gains an optional `download_url:
  string | null` so other consumers can opt in without changes.

## 2. QnA Notifications: Owner Only

- `_iter_staff_recipients` filtered TenantMembership on
  `role__in=["owner", "admin", "teacher", "staff"]`, which routed every
  student QnA into the assistant teacher's phone as well — and they get
  treated as a normal person after hours. The filter narrows to
  `role__in=["owner"]` per the tchul director's explicit request.
- `admin` is intentionally also excluded for now. If a tenant wants to
  add an admin as a secondary recipient, that is a separate follow-up;
  the default needs to be conservative.
- The fixture in `test_qna_notifications.py` flips its membership role
  to `owner` so the same case keeps covering the happy path.

## 3. Unified Alimtalk Envelopes — 7 More Triggers Mapped

`TRIGGER_TO_TEMPLATE_TYPE` used to leave eleven triggers unmapped, with
a comment that any tenant wanting them had to register a private Solapi
template. That is the opposite of the "전부 공용 알림톡 통일" SSOT.
Seven of them join the existing four envelopes:

- `exam_scheduled_days_before` / `exam_start_minutes_before` /
  `exam_not_taken` → `TYPE_ATTENDANCE` (same lecture/session/date/time
  context the existing absent_occurred and lecture_session_reminder
  triggers use).
- `assignment_registered` / `assignment_due_hours_before` /
  `assignment_not_submitted` → `TYPE_ATTENDANCE` (same shape — the
  director writes the assignment-specific copy in `#{선생님메모}`).
- `retake_assigned` → `TYPE_CLINIC_INFO` (a retake is logistically a
  remedial clinic — place/date/time).

The remaining four (`video_encoding_complete`, `matchup_report_submitted`,
`qna_answered`, `counsel_answered`) have no honest match in the four
ITEM_LIST envelopes, so they stay explicitly **보류** with a backlog
pointer. The comment block at the bottom of `TRIGGER_TO_TEMPLATE_TYPE`
records why each one was left out so the next person doesn't quietly
add them to the wrong envelope.

## 4. Clinic Session Destroy: clinic_cancelled

`SessionViewSet.perform_destroy` used to cascade-delete participants in
silence. Students and parents who had a reservation lost it without a
single Alimtalk. We now snapshot every PENDING/APPROVED participant
before the delete, build the same `_status_notification(CANCELLED)`
event the per-participant cancel endpoint emits, and dispatch them on
`transaction.on_commit` so the DB delete and the notifications stay
consistent. If the notification dispatch fails, the database delete
still succeeds — observed in the existing `_send_clinic_notification`
exception handler.

## 8. Residual Observations

- **AI worker cold start (5–10 min)** is the underlying UX friction
  behind "매치업 분석중이라면서 이미지가 안 뜸". Showing the image
  immediately solves the operator pain, but a warm-pool or
  pre-provision policy for the AI worker ASG is a separate decision
  (cost vs. response time) parked in `project_open_backlog`.
- **Four 보류 alimtalk triggers** (video / matchup / qna_answered /
  counsel_answered) still rely on tenant-registered Solapi IDs.
  Returning to them needs a director call on whether to mint a new
  envelope or use an out-of-band channel; not done in this release.
- **E2E `QnA 왕복` flake** is unresolved in this release. The Cloudflare
  deploy went green, so the page is live; the test is treated as a
  flake until a second consecutive failure appears.
- **`admin` membership role** is excluded from QnA recipients along
  with teacher and staff. If a tenant operates with multiple owners /
  director-style admins, they will need to be added by name —
  intentional conservative default.
