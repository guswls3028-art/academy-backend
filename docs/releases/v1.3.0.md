# V1.3.0 Release Notes

**Status:** Production-deployed / GO  
**Release date:** 2026-06-12 KST  
**Scope:** Production audit risk closure, API edge HTTPS root fix, video observability, messaging template safety, E2E reliability

This release seals the 2026-06-12 production domain audit. The main operational
risk was the public API edge intermittently timing out behind Cloudflare over an
HTTP-only origin. The root path is now fixed at the infrastructure layer, with
frontend retry kept only as defense-in-depth.

## 1. Decision

GO for current production operation.

The release is production-proven for API edge, tenant/program public routing,
video readiness, messaging effective-template safety, cleanup residue handling,
tools/matchup E2E reliability, and standard backend/frontend deployment gates.

## 2. Change Summary

Infrastructure:

- Issued ACM certificate for `api.hakwonplus.com`, added ALB HTTPS:443 listener,
  and moved Cloudflare to Strict/DNS-only origin access.
- Added SSOT/deploy support for the API HTTPS listener and video observability
  thresholds.
- Added API-container read-only IAM for video readiness diagnostics.
- Ensured the five expected `academy-video-*` CloudWatch alarms, including
  `academy-video-UploadFailures`.
- Reclassified disabled low-queue scaling-control alarms as informational in
  ops healthcheck.
- Updated DB connection budget docs to match production: `db.t4g.large`,
  API desired 2, AI desired 0, and `DB_CONN_MAX_AGE=0`.

Backend:

- Added effective Alimtalk template resolution so unified/common templates are
  reflected in serializers, preflight, stale-template audit, and operations
  status.
- Blocked enabling AutoSendConfig rows unless the effective template is
  approved; migrated existing unready enabled rows off.
- Expanded E2E cleanup to fee templates, deleting unreferenced residue and
  deactivating referenced residue.
- Isolated matchup optional-module import tests from broad-suite pollution.

Frontend:

- Public `hakwonplus` `/landing*` now routes safely to `/promo` except landing
  admin routes.
- Promo guard no longer blanks when the public program API is unavailable.
- Auto-send settings show effective template status/source.
- Production E2E auth now backs off on 429 throttles; tools native dialog and
  matchup storage tab selectors were updated to current UI behavior.
