# V1.2.39 Release Notes

**Status:** production-deployed and verified
**Release date:** 2026-06-07 KST
**Scope:** Homework submit/grade real-use chain, lecture cleanup delete guard, and student-domain launch-readiness SSOT update

This release closes the homework full-chain launch gap from v1.2.38. The
student-domain broad public launch decision remains HOLD because account
recovery activation, OMR upload/match/review, clinic remediation, and
beginner/misuse behavior still need full-chain proof or documented launch
exceptions.

## 1. Decision

Controlled/internal expansion remains **GO**.

Broad public promotion or large external expansion remains **NO-GO / HOLD**
until the remaining launch gates are passed or explicitly accepted as documented
exceptions:

- account-recovery activation browser chain;
- OMR upload -> match/review -> grading -> admin score board -> student result;
- failed result -> clinic target -> clinic booking/attendance -> remediation;
- beginner/misuse behavior suite across double-submit, stale tabs, mobile
  keyboard, back/forward, and repeated role switching.

## 2. Fix Summary

- Added `frontend/e2e/student/homework-submission-realuse.spec.ts`, a production
  real-use canary for homework assignment, student file submission, admin
  grading, student grades projection, and cleanup.
- Updated the lecture/session delete guard so a homework that has already been
  removed from its session no longer blocks cleanup deletion of generated
  E2E sessions and lectures.
- Kept history-preserving blockers intact: homework scores, submissions,
  attendance, enrollments, videos, and other operational history still block
  deletion through the existing guard paths.
- Added backend regression coverage for deleting a session/lecture after a
  homework has been removed from the session.
- Updated launch-readiness, Phase 2 audit, and frontend real-use E2E inventory
  docs so the homework create/submit/grade chain is no longer listed as an
  open gap.
- Refreshed deploy verification reports and history snapshots after the backend
  deployment.

## 5. Residual Risk

No known critical/high bug remains in the homework submit/grade or touched
lecture cleanup surface. Remaining launch risk is outside this release scope:

stress suite.
