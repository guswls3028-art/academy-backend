**Status:** Production-deployed / GO

This release turns the 2026-06-12 production audit board into durable release
gates. The intent is preventive: catch hidden infra, video, messaging, billing,
E2E residue, and production edge regressions before a user reports them.

## 2. Change Summary

Backend:

- Added `python manage.py production_canary`, a read-only tenant invariant check
  for tenant/program health, effective Alimtalk template readiness, messaging
  worker heartbeat, video HLS/job consistency, E2E residue, and billing guards.
- Hardened video canary coverage for stale `QUEUED`/`RUNNING`/`RETRY_WAIT` jobs
  and mismatched `Video.current_job` tenant/video links.
- Added regression tests for production canary risk classes.
- Added `scripts/v1/run-production-canary.ps1` for HTTP, AWS, SQS/DLQ,
  CloudWatch, Batch, and remote Django checks.
- Added SSM remote execution helper with base64/stdin command delivery so shell
  quoting does not corrupt future remote Django commands.
- Changed API ASG SSM targeting to healthy InService instances only.
- Documented the canary runbook in `docs/operations/production-canary.md`.

Frontend:

- Added `e2e/smoke/production-canary.spec.ts` for production API health,
  `/promo`, admin login, and student login smoke.
- The canary fails on production request failures and critical 4xx/5xx document,
  script, stylesheet, fetch, XHR, or API responses.
- Added `pnpm test:e2e:canary`.
- Inserted the canary into the post-deploy Cloudflare Pages quality gate before
  tenant availability and roundtrip flows.
- Increased post-deploy E2E timeout to keep the existing high-signal checks from
  being crowded out.
