# V1.9.0 Release Notes

**Status:** Production-deployed / GO with an explicit external Hancom integration boundary

**Release date:** 2026-07-20 KST

**Scope:** Problem Studio AI source transcription, sealed Windows Hangul
companion distribution, one-time document handoff, and production real-use
closure

## 1. Decision

GO for production use of the staff Problem Studio upload, AI transcription,
review-package download, HWPX generation, Windows companion installation, and
one-time handoff flow.

The operational path no longer depends on an unpublished local script or an
unconfigured OpenAI secret. Staff can obtain the exact companion package from
the Tools screen, and the AI worker uses the model-scoped AWS Bedrock Nova 2
Lite inference profile when no OpenAI key is configured.

## 2. Business and UX Contract

- The staff Tools tab exposes `AI 시험지 타이핑`. Source files remain local
  until the user explicitly starts a job; completed output is not downloaded
  automatically.
- The result panel always exposes `Windows 연결 프로그램 설치`, explains the
  one-time current-user install, and exposes `검수본 ZIP 내려받기` and
  `한글에서 열기` only after a completed transfer job.
- The self-contained Windows x64 companion copies itself to
  `%LOCALAPPDATA%/Academy/HangulCompanion`, registers `academy-hangul://` under
  HKCU, and requires no administrator permission.
- A five-minute opaque handoff can be consumed once. The companion downloads a
  bounded result ZIP, verifies the server-declared size and SHA-256, requires
  exactly one bounded `03_자체양식_문제검수본.hwpx`, and never saves, closes, or
  exits the user's Hangul process.
- Direct insertion is attempted only into an already-visible normal-edit-mode
  Hangul document. Otherwise the companion opens the extracted HWPX as a new
  document.

## 3. Security, Tenant, and Provider Contract

- The companion endpoint requires an authenticated active staff membership,
  verifies the sealed R2 object size and SHA-256 metadata before presigning,
  returns no R2 key, uses a ten-minute URL, and sends `Cache-Control: no-store`.
- Result objects and temporary source archives remain tenant-prefixed. The
  worker rejects tenant/payload mismatches and deletes the source archive on a
  terminal path.
- Release companion builds accept handoffs only from HTTPS
  `api.hakwonplus.com`; an arbitrary-HTTPS-host finding discovered during the
  security audit was fixed before the sealed v1.0.0 artifact became reachable.
- The production AI worker has no OpenAI key. It therefore uses
  `global.amazon.nova-2-lite-v1:0` through the EC2 instance role. IAM grants only
  `bedrock:InvokeModel` for that inference profile and foundation model. The
  existing bounded unit limit, tenant AI quota, 6,000-token output cap, request
  timeouts, and per-unit local OCR fallback remain enforced.

## 6. External Hancom Boundary

The verification workstation has no Hancom/HWP COM registration and no HWP or
HWPX file association, so a licensed Hancom window could not be used to prove
the final COM `InsertFile` action. The production handoff, one-time consume,
download, size/SHA-256 verification, and HWPX extraction are covered by the
packaged executable's diagnostic mode. Commercial direct-insert distribution
still requires Hancom Automation approval/security-module registration and a
trusted Windows code-signing certificate; this build is unsigned because no
certificate is installed. These external prerequisites do not block ZIP/HWPX
download and manual review, but they remain explicit gates for a trusted
commercial native-insert experience.

## 7. Final State

Problem Studio is available to active staff in production with a bounded AI
transcription path, explicit download semantics, sealed companion delivery,
tenant-safe one-time handoff, and local OCR resilience. No release-scoped
Critical or High software finding remains. The only unproven surface is the
external licensed Hancom COM insertion described above.
