# V1.2.38 Release Notes

**Status:** production-deployed and verified
**Release date:** 2026-06-07 KST
**Scope:** Student launch canary seal, signup Alimtalk dedupe, OMR/video real-use proof, and launch-readiness SSOT update

This release closes the highest-risk student-domain launch-readiness follow-up
after v1.2.37. It upgrades the previous controlled-recipient blocker into a
passed production canary, hardens the E2E cleanup paths that can create user
noise, and records the current GO/NO-GO decision in the launch SSOT.

## 1. Decision

Controlled/internal expansion is **GO**.

Broad public promotion or large external expansion remains **NO-GO / HOLD**
until the remaining full-chain launch gaps are either passed or explicitly
accepted as documented launch exceptions:

- account-recovery activation browser chain;
- OMR upload -> match/review -> grading -> admin score board -> student result;
- failed result -> clinic target -> clinic booking/attendance -> remediation;
- homework create -> student submit -> admin grade -> student result;
- beginner/misuse behavior suite across double-submit, stale tabs, mobile
  keyboard, back/forward, and repeated role switching.

## 2. Fix Summary

- Dedupe registration approval and welcome account notices by normalized
  recipient so the same student/parent phone receives one owner-template
  Alimtalk, not duplicate student+parent notices.
- Hardened the production signup approval canary:
  - exact controlled recipient only;
  - real-send retries disabled;
  - Tenant 1 auto-approve setting is temporarily forced to manual approval and
    restored in `afterAll`;
  - unexpected direct-create response records the created student id for cleanup.
- Changed score-report real-use cleanup to use bulk delete/permanent delete for
  generated students, avoiding withdrawal-notification side effects from detail
  delete.
- Fixed the mobile narrow viewport spec for ESM `__dirname`.
- Repaired the OMR tenant real-use recovery fixture so blank-answer recovery
  tests do not call grading through an invalid zero-answer submission path.
- Extended the student video playback post-deploy smoke to accept the canonical
  public video session returned by `/student/video/me/`.
- Updated launch-readiness, Phase 2 audit, and frontend real-use E2E inventory
  docs with the actual production evidence and current launch decision.
