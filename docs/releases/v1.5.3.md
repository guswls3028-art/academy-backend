# V1.5.3 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-06-28 KST
**Scope:** Tenant 2 matchup question-crop incident closure

This patch seals the production fix for matchup question crops where clean-PDF
materials could appear incomplete: wide questions were clipped back to one
column, and crop padding could pull in the next question on two-column pages.

## 1. Decision

GO for production use.

Tenant 2 document `303` was reanalyzed on production after deployment and the
current 83-question output was visually audited across all crop sheets and the
real admin matchup page. The reported wide-question half-crop class and the
follow-up same-column bleed class are closed for this workflow.

## 2. Change Summary

- Preserved already-wide matchup bboxes during upload padding so full-width
  clean-PDF questions are not clipped back into a single document column.
- Capped crop padding before the next question top when the next question is in
  the same vertical reading flow.
- Changed next-question detection to ignore the opposite column on two-column
  pages, preventing side-by-side questions from disabling the same-column cap.
- Added regression tests for wide bboxes, normal column bboxes, same-column
  next-question caps, and full-width next-question caps.
- Reanalyzed production document `303` and verified the regenerated 83-question
  output visually.
