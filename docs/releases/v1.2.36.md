# V1.2.36 Release Notes

**Status:** production-deployed and verified
**Release date:** 2026-06-07 KST
**Scope:** Student-domain launch stability seal, production API redirect recovery, real-use E2E and Alimtalk proof

This release seals the student-domain launch-readiness pass before broad
promotion/expansion. It treats the student account graph as the product center:
signup, account recovery, teacher/admin password reset, OMR/results, clinic,
homework, video, QnA, and tenant isolation all have to prove together.

## 1. Root Cause

- Student identity, parent/student accounts, password delivery, and downstream
  content flows had been improved in slices, but the release boundary did not
  yet have one SSOT document plus a production proof bundle.
- Production API requests were blocked by a Django HTTPS self-redirect loop
  behind Cloudflare/ALB when proxy scheme metadata was not preserved. This
  caused authenticated Playwright production E2E requests to fail before the
  student flows could be tested.
- The account-recovery E2E spec modeled limglish production as
  `hakwonplus.com/login/limglish`; production tenant resolution is host-based,
  so the correct real path is `https://limglish.kr/login`.

## 2. Fix Summary

- Added `docs/domain/student-core.md` as the student-domain SSOT and aligned the
  launch-readiness/refactor validation docs around account, OMR/results,
  clinic, homework, video, QnA, and tenant-isolation gates.
- Hardened student identity/password/account recovery tests and the
  submission/result scope boundary.
- Added `submissions.selectors` for scoped answer-map reads and routed result
  services through it.
- Frontend student/account surfaces were aligned for signup, account recovery,
  teacher/admin password reset, Excel import, account modals, and production
  real-use E2E inventory.
- Changed production Django `SECURE_SSL_REDIRECT` default to `False` because
  HTTPS is enforced at the edge/ALB; secure cookies and HSTS remain enabled.
  Added a smoke regression so prod defaults do not reintroduce the API redirect
  loop.
- Updated account-recovery E2E so local/preview still use `/login/{tenant}`,
  while production limglish uses the real tenant host.

## 5. Residual Risk

- The physical phone screen cannot be inspected from the service side. The
  project SSOT defines operational Alimtalk success as provider-backed
  `NotificationLog.status=sent` with `provider_message_id`; visible body copy
  remains verifiable only from the controlled recipient device.
- The exam data-flow canary intentionally skips direct answer submission when
  the student has no accessible questions for the chosen production exam.
