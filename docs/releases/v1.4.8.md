# V1.4.8 Release Notes

**Status:** Production-deployed / GO
**Release date:** 2026-06-24 KST
**Scope:** Problem Studio source transfer + beta rewrite workflow

This patch turns Problem Studio into a practical first-run source transfer tool.
The default workflow is deliberately conservative: teachers upload official or
reference material, receive editable Hangul-compatible draft documents, and
finish any educational judgment manually. AI rewrite remains a secondary beta
panel for review candidates only.

## 2. Change Summary

Backend:

- Added the canonical source-transfer endpoint:
  `POST /api/v1/tools/problem-studio/transfer-document/`.
- Kept beta rewrite on the async job path:
  `POST /api/v1/tools/problem-studio/jobs/` and
  `GET /api/v1/tools/problem-studio/jobs/<id>/`.
- Removed the legacy synchronous generate endpoint to avoid duplicate Problem
  Studio surfaces.
- Added PDF, HWP, HWPX, DOCX, ZIP, and image transfer handling for editable
  `.doc` package output.
- Added text-only HWP and nested ZIP extraction for beta fallback without
  running expensive image normalization.
- Moved PDF rendering through the PyMuPDF adapter to keep domain code away from
  direct infrastructure imports.
- Added destructive tests for oversized ZIP structures, transfer package member
  limits, and malformed JSON payloads.
- Updated `docs/domain/problem-studio.md` with product scope, routes,
  operating limits, and next investments.

Frontend:

- Re-centered the UI on one primary CTA: `원본 한글로 저장`.
- Accepted real teacher source formats in the picker copy and file contract:
  PDF, HWP, HWPX, DOCX, ZIP, and images.
- Collapsed AI rewrite behind `Beta 재작성 / 검수용 후보 생성`.
- Removed large planning/backlog/phase UI that made the first workflow feel
  heavier than it is.
- Added optional beta modes for similar type, trap reinforcement, and concept
  explanation candidates.

## 5. Product Review

The most sellable positioning is "교재 원본 이관 + 편집 초안 자동 생성".
That solves the immediate pain: no retyping, fewer copy/paste misses, and
preserved diagrams/tables/images for teacher review.

The beta rewrite should stay visually secondary until OCR/template fidelity and
human review controls are stronger. In customer language, it is a fast idea
generator, not a final-authoring engine.

Next best investments:

- OCR for scanned PDFs/images so beta can read photocopied or raster-only
  source pages.
- Native HWPX output or a stricter academy template renderer for better Korean
  document fidelity.
- A short review checklist in the exported draft package so teachers know what
  must be checked before classroom use.
- Per-source preview and selective export if teachers start uploading very
  large publisher packages.

## 6. Residual Risk

- Scanned PDFs/images transfer visually but do not feed beta text extraction
  until OCR is added.
- Current editable output is Hangul/Word-compatible `.doc` HTML, not native
  binary `.hwp`.
- Beta rewrite is rule/AI candidate generation and must remain teacher-reviewed.
- CI logged the existing missing `E2E_STUDENT_USER/PASS` student chain secret as
  a skipped optional smoke path.
- AI ASG is intentionally scaled to zero, so CI skipped AI instance refresh.
