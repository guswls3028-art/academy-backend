# V1.2.8 Release Notes

**Status:** production-deployed  
**Release date:** 2026-05-23 KST  
**Scope:** Attendance roster write SSOT + local PG-only concurrency test guard

## 1. Attendance Roster Write Convergence

- Added `apps.domains.attendance.services.create_attendance_roster()`.
- Added `ensure_session_roster_membership()` as the shared unit for:
  - enrollment activation/reactivation,
  - auto-assigned fee reactivation,
  - `SessionEnrollment` creation,
  - `Attendance` creation with idempotent `get_or_create`.
- `AttendanceViewSet.bulk_create` is now a transport facade over the attendance service.
- `bulk_create_session_enrollments()` now uses the same roster membership helper, so attendance roster creation and session-enrollment creation cannot drift on fee/status/attendance side effects.
- Malformed `session` and `students` payload IDs now fail with 400 instead of reaching ORM conversion errors.

## 2. Test Harness Cleanup

- The fees PostgreSQL row-locking concurrency test is now explicitly skipped on non-PostgreSQL local smoke databases.
- This preserves the real PG concurrency test contract while keeping the default SQLite smoke suite green and meaningful.
