# V1.4.3 Release Notes

**Status:** Production-deployed / GO  
**Release date:** 2026-06-21 KST  
**Scope:** Tenant 2 dense workbook owner-row segmentation seal

This patch closes the remaining non-`26-1m` physical misses found after the
Tenant 2 full-corpus audit. The main failure class was dense workbook pages
where one large physical question owns many numbered subrows. The splitter now
keeps those subrows inside the parent problem instead of cutting them as fake
top-level questions.

## 1. Decision

GO for current production operation.

For Tenant 2 scan/PDF materials in the audited corpus, manual physical GT recall
is now 1.0 with physical_missed 0. Hand-shot photos remain explicitly out of
scope.

## 2. Change Summary

Backend:

- Added dense owner-stem detection for workbook pages with a large main question
  followed by numbered fill-in or O/X rows.
- Blocked the dense fill-in-row fast path when a parent main stem owns the
  numbered rows.
- Filtered plain numbered subrows under detected parent stems before region
  construction.
- Covered the low-number reset case where Q8/Q9 visual O/X questions own
  internal rows `1..6` or `1..8`, even when dispatcher first-pass splitting uses
  `prefer_marginal=False`.

Frontend:

- Not changed.
