# V1.2.2 Release Notes

**Status:** sealed
**Seal date:** 2026-05-22
**Scope:** tenant/login/account recovery SSOT, sensitive log hardening, refactor documentation alignment

## 1. Account Recovery

- Public login account recovery is centralized on `/api/v1/auth/account-recovery/dispatch/`.
- Username recovery sends the account ID to the verified phone without changing the password.
- Password recovery creates a `PendingPasswordReset` hash and keeps the old password valid until the delivered temporary password is used at login.
- Temporary passwords are 8 numeric digits for parent usability.
- Delivery failure restores the previous pending reset, or clears the new pending reset if none existed.
- Disabled/test tenant messaging no longer enqueues owner-template Alimtalks and does not create undisclosed pending passwords.

## 2. Login And Auth Safety

- Pending temporary passwords are checked before mutation and consumed only after the account is confirmed loginable.
- Inactive/non-loginable users cannot consume a pending reset.
- Staff-only password reset options now use project `TokenVersionJWTAuthentication`, so stale/revoked staff tokens cannot use `temp_password` or `skip_notify`.

## 3. Sensitive Data Handling

- `NotificationLog.message_body` no longer stores account/credential notification bodies.
- Existing credential-like log bodies are redacted by migration.
- Student registration no longer persists `initial_password_plain`; existing values are cleared by migration.
- Registration approval messages use a non-secret phrase for the student password field.

## 4. Frontend

- Login page account recovery UI now targets the canonical recovery endpoint.
- Account recovery modal has focus trapping, focus restore, background inerting, and live-region status/error feedback.

## 5. Documentation

- Deployment docs were aligned with the current GitHub Actions workflow: build, migration, service deploy, verify.
- Video Batch runbook was rewritten around current `scripts/v1/resources/*` and removed legacy `scripts/infra/*` references.
- Account recovery SSOT documents pending reset semantics, disabled tenant behavior, and log redaction.

## 6. Verification Targets

- Backend account recovery/password safety/registration safety tests.
- Messaging log redaction tests.
- Frontend typecheck/lint/build and account recovery E2E.
- Local product gate and production Alimtalk verification before final seal.
