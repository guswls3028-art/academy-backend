## 2. Student Score Progression Contract

- `GET /results/admin/student-grades/?student_id=<id>` now returns
  `exam_trend` and `exam_summary` in addition to the existing exam/homework
  lists. New scored regular exams appear automatically without a batch job or
  manual aggregation step.
- One trend point represents the latest eligible representative `Result` for a
  distinct regular exam. Retakes of the same exam do not create duplicate
  rounds; different regular exams are ordered by their academic session date
  and order, with deterministic recorded-time fallbacks.
- Scores with different maxima are normalized to percentages. Zero is a valid
  score, bonus scores above 100% are preserved, and negative, non-finite, or
  non-positive-maximum rows are excluded from analytics and strict JSON output.
- `NOT_SUBMITTED` remains visible in the detailed list as `미응시`, but is not
  converted into zero and is excluded from trend, average, and pass-rate
  denominators.
- Archived regular exams remain in student history and are marked as archived.
  System lectures are excluded. Multi-lecture exams use only the lecture of the
  result enrollment, preventing metadata or visibility policy from being
  borrowed from another lecture.

## 3. Admin and Teacher UX

- Added a shared responsive line chart to the admin student detail overlay and
  teacher mobile student detail page.
- The chart shows cumulative rounds, latest/average/best score, previous-round
  and first-to-latest changes, raw-score tooltips, archive/retake context, and
  lecture filters. Long histories scroll horizontally instead of compressing
  labels, and filter selection exposes `aria-pressed` to assistive technology.
- Loading and API failure states are explicit. A failed score request is shown
  as unavailable with retry actions, never as a real zero-count student.
- Teacher and admin rows render `NOT_SUBMITTED` neutrally. Homework cards use
  the backend composite identity so reused homework rows reconcile stably.

## 4. Tenant, Security, and Analytics Hardening

- Student, enrollment, exam, session, lecture, homework, clinic-link, and
  attempt metadata reads fail closed through the request tenant.
- Generic result attempt links are trusted only when the attempt matches the
  exact enrollment/exam pair. Corrupt foreign targets, attempts, enrollments,
  sessions, and homework relations cannot leak metadata or inflate retry counts.
- Session lookup is bulk-prefetched by `(exam, enrollment lecture)`, removing
  the former per-exam lookup pattern while retaining constant query growth.
- Enterprise analytics now excludes only structured synthetic prefixes such as
  `[E2E-*]`, `LOCAL-DEMO`, and `DEMO-*`. Legitimate operational names such as
  `Ymath 주간 테스트` and `Level Test` remain in analytics.
