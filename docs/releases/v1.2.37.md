# V1.2.37 Release Notes

**Status:** production-deployed and verified
**Release date:** 2026-06-07 KST
**Scope:** Student active-enrollment SSOT, fail-closed projections, and real-use E2E cleanup hardening

This release closes the Phase 2 student-domain stability pass. It does not mark
the broad promotion/expansion launch as ready. The code, docs, deploy, and
production gates are sealed; the remaining launch blocker is the controlled
signup/Alimtalk canary fixture collision documented below.

## 1. Root Cause

- Student-facing content projections were still scattered across exam, result,
  homework, attendance, video, dashboard, wrong-note, and attempt-history paths.
  That made inactive enrollments and stale IDs a recurring bug source.
- Direct student access to an inaccessible exam detail URL could remain in a
  loading/retry state instead of failing closed quickly.
- E2E retry behavior could leave a test-only counsel post behind if the first
  attempt created the post and the next admin-page navigation failed before
  cleanup.
- The student-domain SSOT and launch-readiness docs needed to reflect actual
  implementation and production evidence, not aspirational structure.

## 2. Fix Summary

- Added canonical active-enrollment selectors:
  `active_enrollments_for_student()` and
  `active_enrollments_for_students()`.
- Routed student-facing exam list/detail/submission, result detail, grades
  summary, wrong-note/PDF/status, exam-attempt history, attendance summary,
  session-hide mutation, dashboard scoped notices, video list/stats, and
  student video access context through the canonical active-enrollment boundary.
- Preserved the public/system video exception only through explicit
  `include_system=True`.
- Rebased the refactor-boundary snapshot after reducing internal cross-domain
  imports and documenting the new public selector edge.
- Added guarded signup-approval E2E, student exam 4xx fail-closed behavior,
  current UI assertions for counsel/homework/scores/video flows, retry-safe exam
  cleanup, production navigation timeout hardening, and retry-safe counsel
  cleanup.
- Updated `student-core.md`, launch-readiness, Phase 2 audit, and deploy report
  artifacts with the verified SSOT and evidence.

## 5. Residual Launch Blockers

- Production signup approval + controlled Alimtalk canary is intentionally not
  executed because controlled recipient `01031217466` is already attached to
  active Tenant 1 E2E fixtures: student `1890` and student `1201`. Do not use
  an arbitrary phone number to bypass this.
- Account-recovery activation still needs a browser spec that proves
  temporary-password login activates the pending reset. Backend and modal tests
  cover the safety rules.
- Full OMR upload/match/review -> grading -> student result, clinic remediation,
  homework create/submit/grade, video playback/progress persistence, and mobile
  visual review remain launch-gate work. Current production render/API and
  backend regression coverage passed, but these chains are not all promoted to
  complete browser canaries.

## 6. Decision

Student active-enrollment projection bugs addressed in this release are sealed
and deployed. Broad promotion/expansion launch remains no-go until the residual
launch blockers above are closed or explicitly accepted as documented launch
exceptions.
