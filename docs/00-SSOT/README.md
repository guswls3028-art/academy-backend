## 원칙

- **SSOT(실행)**: 인프라 파라미터는 `params.yaml`을 기준으로 한다.
- **SSOT(배포/CI 문서)**: 배포 아키텍처는 `v1.2.0/RELEASE-NOTES.md`(인프라 변경 모음) + `v1.1.0/`(직전 봉인본)을 함께 본다.
- **SSOT(도메인)**: 메시징/OMR/커뮤니티/운영정책 등 도메인 본문은 `v1.1.1/`이 정본. V1.2.0에서 변경된 부분은 RELEASE-NOTES에 명시.
- **봉인 버전**: 아카이브된 버전 문서는 수정하지 않는다.
- **충돌 시**: 실행 코드(scripts, workflows, Dockerfile) > SSOT 문서 > 기타 문서.
