# Limglish·운영 요구사항 설계 체크리스트

> 선생님·운영 논의(Limglish 등) 반영. **환경 제공** 우선 — 구체 수치(예: 클리닉 30분 전)는 하드코딩하지 않고, 사용자가 템플릿·타이머로 커스텀 가능하게 한다.

---

## 1. 회원가입 필드 정책

| # | 요구사항 | 상태 | 비고 |
|---|----------|------|------|
| 1.1 | 주소를 학생에게 **굳이** 필수로 두지 않음 | ✅ 반영 | DB/모델은 optional 유지 |
| 1.2 | 회원가입 시 **모든 필드를 필수로 채우게** (계열 제외) | ✅ 반영 | 계열만 선택, 나머지 필수 입력 |
| 1.3 | 출신중학교, 주소 — 폼에 포함·필수 입력 | ✅ 반영 | “모든 필드 필수(계열 제외)”에 포함 |
| 1.4 | 성적 문자 연동 | 🔲 환경 | 자동발송 트리거·템플릿으로 사용자 설정 가능 (아래 2.x) |

**구현 포인트**
- Backend: `RegistrationRequestCreateSerializer` — 계열(`major`) 제외한 필드에 대해 빈 문자열 불허(또는 required=True).
- Frontend: 회원가입 모달 — 계열 제외 모든 입력란 필수 표시(*) 및 제출 전 검증.

---

## 2. 메시지·자동발송 환경 (타이머·트리거)

| # | 요구사항 | 상태 | 비고 |
|---|----------|------|------|
| 2.1 | **출결** — 학원에서 이미 자동화 | 🔲 참고 | 기존 출결 알림 유지, 트리거만 확장 시 노출 |
| 2.2 | **강의** — 30분 전 등 | 🔲 환경 | “30분 전” 고정 X, **타이머 설정 가능** (기본값·변경 가능) |
| 2.3 | **클리닉** — 1시간 전, 장소·시간 등 최소 정보 | 🔲 환경 | 타이머 설정 + 템플릿 변수(장소·시간)로 사용자 정의 |
| 2.4 | **넉넉한 base** 템플릿 | ✅ 유지 | 기존 템플릿 구조·카테고리 유지 |
| 2.5 | **타이머 설정 가능** (기본값 셋팅 및 변경 가능) | ✅ 반영 | AutoSendConfig에 `minutes_before` 등 추가, UI에서 설정 |
| 2.6 | 과제·시험 완성 알림 | 🔲 환경 | 트리거 추가 시 선택 가능하도록 확장 |

**구현 포인트**
- `AutoSendConfig`: `minutes_before` (nullable, 정수) — “이벤트 N분 전” 발송용. 트리거별 기본값은 UI/설명으로만 안내.
- 트리거 확장: `lecture_reminder`, `attendance_*`, `exam_submitted` 등 필요 시 추가. **실제 발송 스케줄러(cron/celery)** 는 별도 작업(환경만 제공).
- 프론트: 자동발송 설정 화면에 “N분 전 발송” 입력란 추가(해당 트리거만 표시).

## 3. 장소·본부·다캠퍼스

| # | 요구사항 | 상태 | 비고 |
|---|----------|------|------|
| 3.1 | 같은 수업이 **요일별 다른 장소**(예: 목요일 두각, 토요일 대치메카) | 🔲 설계 | Session(강의 차시)에 location/venue 필드 또는 FK |
| 3.2 | 학생앱에서 **즉각 장소 확인** | 🔲 구현 | 일정/세션 상세에 장소·전화 노출 |
| 3.3 | 두각·대치메카 등 **전화번호 등록 및 네비게이팅** | 🔲 구현 | 장소별 전화·주소·링크(지도) |
| 3.4 | **본부** 전화번호 중요 (실장님 전화받는 곳) | ✅ 반영 | Tenant에 `headquarters_phone` (본부 전화) 추가 |
| 3.5 | 학생앱 홈 하단 — **본부 진입게이트 정보** | ✅ 반영 | 대시보드 하단에 본부(학원명·전화) 블록 |

**구현 포인트**
- Tenant: `headquarters_phone` (CharField, blank=True). 기존 `phone`은 그대로 두고, 본부용으로 구분.
- 학생 대시보드 API: `tenant_info: { name, phone, headquarters_phone }` 등 반환.
- (추후) Session location 확장 시: `location`, `location_phone`, `location_map_url` 등.

## 5. 구현 완료·검증·배포

| 단계 | 내용 |
|------|------|
| 설계 체크리스트 | 본 문서 |
| 구현 | 1) 회원가입 필수(계열 제외) 2) 자동발송 타이머(minutes_before) 3) 본부 전화·학생앱 본부 블록·설정 내 소속 학원 카드 |
| 검증 | 아래 검증 체크리스트 실행 |
| 배포 | 기존 deploy.ps1 / ECR 반영 절차 사용 |

### 검증 체크리스트

- [ ] **회원가입**: 계열 제외 모든 필드 비어 있으면 제출 시 백엔드/프론트에서 에러. 출신중학교·주소·휴대전화·성별·반·학년·고등학교명(또는 중학교명) 필수.
- [ ] **자동발송**: 설정 > 메시지 > 자동발송에서 트리거별 "N분 전" 입력란 표시·저장·재조회 시 반영.
- [ ] **본부**: Tenant에 `headquarters_phone` 저장 가능. 학생앱 홈 하단 "본부" 블록에 학원명·본부 전화 노출. 설정 > 내 정보 > 소속 학원에서 본부 전화번호 편집(owner만).
- [ ] **마이그레이션**: `python manage.py migrate` 로 messaging 0003, core 0008 적용.

**배포 반영**: 백엔드 마이그레이션 적용 후 API/워커 재기동 (`backend/deploy.ps1` 또는 기존 절차). 프론트 빌드·정적 배포 (`deploy-front.ps1` 등). 배포 후 검증 체크리스트를 운영에서 한 번 더 확인 권장.

---

## 6. 참고 — “실제 반영하지 말고 환경만”

- **클리닉 30분 전 알람** 등 구체 수치는 **하드코딩하지 않음**.
- 트리거 + **타이머(minutes_before)** + **템플릿**으로 학원이 직접 “30분 전”, “1시간 전”, “출결 알림” 등을 설정할 수 있도록 한다.
