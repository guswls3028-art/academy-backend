# 운영 배포

**리전**: ap-northeast-2 (서울)  
**정본**: [deployment-modes.md](deployment-modes.md), [formal-deploy.md](formal-deploy.md), [../ssot/params.yaml](../ssot/params.yaml).
**원칙**: API — ASG min/desired=2. Messaging — ASG min/desired=1 + SQS alarm 기반 StepScaling. AI — AWS/SQS alarm 기반 ASG StepScaling(평소 0대). Tools — ASG min/desired=1. Video — AWS Batch 전용(daemon ASG 폐기 2026-05-10).
**Video Worker 사양**: standard 인코딩 jobdef VCPU=8 / 16GB / timeout 6h. Batch CE 이름 = `academy-v1-video-batch-ce-200gb`.

> **2026-05-25 확인:** 영상 인코딩은 AWS Batch jobdef SubmitJob으로만 실행한다. EC2 Video daemon/ASG에 `docker run` 하지 않는다. SSOT: `infrastructure/deployment-architecture.md` + `apps/worker/video_worker/README.md` + `infrastructure/video-cron-jobs.md`.

## 1.1 배포 방식

| 방식 | 설명 | 권장 |
|------|------|------|
| **CI 자동 배포** | main push → GitHub Actions가 변경 이미지 빌드·ECR 푸시 → `run-migrations`(필요 시) → `deploy-api`/`deploy-messaging`/`deploy-ai`/`deploy-tools`/`deploy-video` → `verify-deployment`. **push=서버 반영** 자동화. | 일상적인 코드 반영. |
| **수동 정식 배포** | `pwsh scripts/v1/deploy.ps1 -AwsProfile default` — 전체 인프라 Ensure + LT 갱신 + ASG instance refresh. 로컬 빌드 없음. | 인프라 변경·안정 반영·릴리즈용. |

**상세·비교표:** [deployment-modes.md](deployment-modes.md), [formal-deploy.md](formal-deploy.md).

## 2. 로컬 빌드/푸시 금지

운영 이미지는 로컬에서 build/push하지 않는다. `backend/.github/workflows/v1-build-and-push-latest.yml`가 6개 이미지(`academy-base`, `academy-api`, `academy-video-worker`, `academy-messaging-worker`, `academy-ai-worker-cpu`, `academy-tools-worker`)를 linux/arm64로 빌드하고 ECR에 `:latest` + `:sha-*`를 푸시한다.

수동 정식 배포는 이미 ECR에 올라간 이미지를 pull/refresh만 한다:

```powershell
cd C:\academy\backend
pwsh scripts/v1/deploy.ps1 -AwsProfile default
```

---

## 3. API 반영

API는 EC2에 SSH로 들어가 수동 `docker run` 하지 않는다. Launch Template/UserData/ASG가 ECR `academy-api:latest`와 SSM `/academy/api/env`를 사용해 컨테이너를 올린다.

검증:

```powershell
pwsh scripts/v1/run-production-canary.ps1 -Mode PostDeploy -AwsProfile default -WriteReport
pwsh scripts/v1/run-deploy-verification.ps1 -AwsProfile default
```

---

## 4. Workers

Messaging/AI/Tools workers는 각 ASG Launch Template과 `scripts/v1/resources/worker_userdata.ps1`로 기동한다. EC2에서 직접 `docker run` 하지 않는다.

| Worker | ECR | ASG | Queue |
|------|------|------|------|
| Messaging | `academy-messaging-worker` | `academy-v1-messaging-worker-asg` | `academy-v1-messaging-queue` |
| AI | `academy-ai-worker-cpu` | `academy-v1-ai-worker-asg` | `academy-v1-ai-queue` |
| Tools | `academy-tools-worker` | `academy-v1-tools-worker-asg` | `academy-v1-tools-queue` |

---

## 5. Video Batch

Video는 EC2 daemon 컨테이너를 수동 실행하지 않는다. 현재 경로는 API `upload_complete` → `VideoTranscodeJob` → AWS Batch `SubmitJob` → `academy-v1-video-batch-jobdef`다.

```powershell
pwsh scripts/v1/verify-video-batch-connection.ps1 -Profile default
```

상세 절차는 [runbooks/video-batch.md](runbooks/video-batch.md)를 우선한다.

## 9. 로컬에서 배포 DB 사용

로컬에서 `python manage.py shell`, `manage.py migrate`, `fix_qna_orphan_created_by` 등을 실행할 때 **배포(프로덕션) RDS/Redis**를 쓰려면, SSM에 저장된 API env를 로컬 `.env`에 반영하면 됩니다.

| 목적 | 명령 (로컬 PowerShell) |
|------|-------------------------|
| **배포 DB/Redis를 .env에 반영** | `pwsh scripts/v1/sync-local-env-from-ssm.ps1 -AwsProfile default` |
| **반영 내용만 확인 (파일 수정 없음)** | `pwsh scripts/v1/sync-local-env-from-ssm.ps1 -AwsProfile default -WhatIf` |

- SSM `/academy/api/env`(JSON)에서 `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_PORT`, `REDIS_HOST`, `REDIS_PORT`, `REDIS_PASSWORD`, `REDIS_DB` 만 읽어와 루트 `.env`에 덮어씁니다. 나머지 키는 그대로 둡니다.
- **주의**: 배포 DB를 쓰면 실제 데이터가 변경됩니다. `migrate`, `fix_qna_orphan_created_by` 등 실행 시 유의하세요.
