# Problem Studio Source Transfer UAT

Last verified: 2026-07-20

목표는 자동 생성 성능 과장이 아니라, 선생님이 실제 받은 자료를 업로드했을 때
`AI 타이핑 시작` 결과물을 내려받아 한글에서 수업용으로 수정할 수 있는지 확인하는 것이다.

## 1. 사전 조건

- 운영 URL: `https://hakwonplus.com/admin/tools/problem-studio`
- 권한: 학원 owner/admin/staff 계정
- 기본 CTA: `AI 타이핑 시작`
- 테스트 파일 예시:
  - PDF 스캔본 또는 출판사 PDF
  - HWP/HWPX/DOCX 원본
  - HWP/DOCX/PDF가 섞인 ZIP
  - 문제 이미지 1~3개

## 2. 합격 기준

| 영역 | 합격 기준 |
|------|-----------|
| 업로드 | PDF/HWP/HWPX/DOCX/ZIP/이미지를 같은 화면에서 등록할 수 있다. |
| 다운로드 | 완료만으로 자동 저장되지 않고 `검수본 ZIP 내려받기`를 눌렀을 때 ZIP이 저장된다. ZIP에는 `00_먼저열기_검수체크리스트.doc`, `01_자체양식_문제검수본.doc`, `02_OCR_연결후보.csv`, `03_자체양식_문제검수본.hwpx`가 포함된다. |
| 열림 | 산출 `.doc` 파일이 한글 또는 Word에서 열리고, `03_자체양식_문제검수본.hwpx`가 한글에서 열린다. |
| 원본 보존 | PDF/이미지 페이지와 HWP 임베디드 이미지가 깨지지 않는다. |
| 구조화 | `00_manifest.json`에 구조화 항목 수, 문제 후보 수, 자동 OCR 처리 단위, 남은 OCR 후보 수, `structure_limit_reached`, 품질 상태가 기록된다. |
| AI 충실도 | AI가 문제를 풀거나 표현을 고치지 않고, 판독 불가능한 부분을 추측하지 않고 표시한다. |
| 연결 프로그램 설치 | 직원 계정만 `Windows 연결 프로그램 설치`에서 봉인된 ZIP을 내려받을 수 있다. ZIP을 푼 뒤 EXE를 한 번 실행하면 관리자 권한 없이 현재 사용자에게 등록된다. |
| 한글 연결 | Windows 연결 프로그램이 있으면 보이는 일반 편집 모드 문서에 삽입하고, 그 외에는 HWPX를 새로 연다. 기존 문서를 저장/닫지 않는다. |
| 수정성 | 선생님이 본문/문항/안내문을 직접 고칠 수 있다. |
| 경고 | 제한된 파일은 조용히 실패하지 않고 체크리스트/리포트에 기록된다. |
| Beta | 재작성은 접힌 Beta 패널에 남고, 기본 이관을 방해하지 않는다. |

## 3. 기본 UAT 절차

1. Problem Studio에 접속한다.
2. 처음 쓰는 Windows PC라면 `Windows 연결 프로그램 설치`를 눌러 ZIP을 내려받고, 모두 푼 뒤 `Academy.HangulCompanion.exe`를 한 번 실행한다.
3. `파일 선택` 또는 드래그 앤 드롭으로 실제 원본 파일을 모두 등록한다.
4. 제목/반명/과목을 실제 수업명으로 입력한다.
5. `AI 타이핑 시작`을 누르고 완료 상태를 확인한다. 이 시점에 자동 다운로드가 없어야 한다.
6. `검수본 ZIP 내려받기`를 누른 뒤 ZIP을 열고 `00_먼저열기_검수체크리스트.doc`를 먼저 연다.
7. `한글에서 열기`를 눌러 보이는 일반 편집 문서의 커서에 삽입되거나, 삽입할 문서가 없으면 HWPX가 새로 열리는지 확인한다.
8. `01_자체양식_문제검수본.doc`에서 자동 분리된 문제/개념 후보를 확인한다.
9. `03_자체양식_문제검수본.hwpx`를 한글에서 열어 텍스트 중심 검수본으로 사용할 수 있는지 확인한다.
10. `02_OCR_연결후보.csv`에서 남은 OCR 후보 ID, 원본, 쪽 범위, 우선순위, 권장처리를 확인한다.
11. `00_변환리포트.html`에서 문서 수, 이미지 수, 경고 수, 구조화 상태, 자동 OCR 처리 수, 남은 OCR 후보 수를 확인한다.
12. `00_manifest.json`에서 `quality_level`, `structured_problem_count`, `ocr_completed_unit_count`, `ocr_pending_unit_count`, `ocr_candidate_count`, `structure_limit_reached`를 확인한다.
13. 각 산출 `.doc`을 한글 또는 Word에서 열어 첫 쪽/중간 쪽/마지막 쪽을 원본과 대조한다.
14. 표, 수식, 그림, 선택지, 정답/해설이 빠지거나 섞인 부분을 체크리스트의 피드백 표에 적는다.
15. 수정 후 수업용 파일로 저장한다.

## 4. 파일 유형별 확인 포인트

| 유형 | 확인 |
|------|------|
| PDF | 페이지 수가 맞는지, 잘림/회전/흐림이 없는지 확인한다. 스캔본은 자동 OCR 텍스트와 원본 이미지를 대조하고, 남은 후보는 CSV로 처리한다. |
| HWP | 본문 텍스트, 임베디드 그림/도표, 표·수식 위치를 원본과 대조한다. |
| HWPX/DOCX | 본문 텍스트 누락이 없는지 확인한다. 표/이미지는 원본 대조가 필요하다. |
| ZIP | 내부 지원 파일이 모두 산출물로 나왔는지 `00_파일목록.csv`와 대조한다. |
| 이미지 | 해상도와 잘림을 확인한다. 자동 OCR 텍스트가 있으면 원본과 대조하고, 없으면 OCR 후보로 남는지 확인한다. |
| DOC | 경고가 떠야 정상이다. DOCX 또는 PDF로 저장 후 재업로드하는 회피 경로를 안내한다. |

## 5. 파괴 테스트

| 케이스 | 기대 결과 |
|--------|-----------|
| 지원하지 않는 확장자만 있는 ZIP | 지원 파일 없음 경고가 리포트에 기록된다. |
| 내부 파일 수가 제한을 넘는 ZIP | ZIP 해제 오류 문서 1개와 경고가 생성된다. |
| 깨진 PDF/HWP/DOCX | 해당 원본의 오류 리포트가 생성되고 전체 요청은 가능한 범위에서 완료된다. |
| 큰 PDF | 60쪽 단위로 `.doc`이 분할된다. |
| 이미지-only PDF | 페이지 이미지는 보존되고, 자동 OCR이 제한 단위 안에서 시도된다. 남은 페이지는 manifest/report/checklist/`02_OCR_연결후보.csv`에 OCR 후보로 기록된다. |
| 80개 초과 구조화 후보 | `structure_limit_reached=true`이고 HWPX 검수 액션에 나머지를 원본 보존 문서에서 확인하라는 안내가 나온다. |
| 만료/재사용 한글 handoff | 5분 만료 또는 1회 소비 뒤 404가 반환되고 새 handoff를 발급해야 한다. |
| 연결 프로그램 객체 누락/크기·SHA 불일치 | 다운로드 API는 URL을 발급하지 않고 503으로 닫힌다. |

운영 자동 진단은 설치 EXE의 아래 명령을 사용한다. 결과 JSON에서 `success=true`, 서버의 `size_bytes`/`sha256` 일치, 추출된 `hwpx_path` 존재를 확인하고, 같은 handoff를 다시 조회하면 404여야 한다.

```powershell
Academy.HangulCompanion.exe --diagnose-handoff "academy-hangul://insert?handoff=..." ".\diagnostic.json"
```

## 6. 구조화 검수

`01_자체양식_문제검수본.doc`와 `03_자체양식_문제검수본.hwpx`는 최종 교재가 아니라 빠른 검수본이다.

1. 상단의 자동 분리 항목 수, 문제 후보 수, 개념 블록 수를 확인한다.
2. 번호가 잘못 끊긴 항목, 표/수식이 본문으로 풀린 항목, 선택지가 누락된 항목을 표시한다.
3. `정답: 검수 필요`, `해설: 검수 후 작성` 항목은 선생님이 직접 채운다.
4. 자동 OCR 텍스트가 있으면 원본 이미지와 대조한다.
5. OCR 후보가 있으면 `02_OCR_연결후보.csv`의 후보 ID와 쪽 범위를 기준으로 시각 보존 합격/수동 전사/비동기 OCR 보강 여부를 결정한다.

## 7. Beta 재작성 검수

Beta는 상품의 기본 약속이 아니다. 아래 기준으로만 확인한다.

1. Beta 패널은 기본 접힘 상태여야 한다.
2. `유사 유형`, `함정 보강`, `개념형` 중 하나를 선택한다.
3. 후보 수를 1~10 사이로 둔다.
4. 결과는 반드시 선생님 검수용 초안으로만 취급한다.
5. 정답/해설이 불확실하면 그대로 배포하지 않는다.

## 8. 피드백 수집 양식

선생님에게 아래 5가지만 받는다.

| 질문 | 답변 |
|------|------|
| 원본을 다시 타이핑하는 시간 대비 얼마나 줄었는가? | |
| 한글에서 바로 열고 수정할 수 있었는가? | |
| 누락/깨짐이 발생한 파일명과 위치는 어디인가? | |
| 결과물에서 가장 먼저 고치고 싶은 불편은 무엇인가? | |
| 유료 기능이라면 OCR 정확도, HWPX 양식 재현, 문제 분리 중 무엇을 먼저 원하나? | |

## 9. 다음 개선 판단

- 스캔본/이미지 자료가 많고 텍스트 수정 요구가 높으면 OCR 정확도와 비동기 대량 OCR을 우선한다.
- 한글 원본 양식 재현 요구가 높으면 HWPX 이미지/표/미주 렌더러와 템플릿 렌더러를 우선한다.
- 선생님이 내용 재구성을 원하면 Beta 후보 승인/거절 흐름을 먼저 만든다.
