# Problem Studio Source Transfer UAT

Last verified: 2026-06-24

목표는 자동 생성 성능 과장이 아니라, 선생님이 실제 받은 자료를 업로드했을 때
`원본 한글로 저장` 결과물을 열어 수업용으로 수정할 수 있는지 확인하는 것이다.

## 1. 사전 조건

- 운영 URL: `https://hakwonplus.com/admin/tools/problem-studio`
- 권한: 학원 owner/admin/staff 계정
- 기본 CTA: `원본 한글로 저장`
- 테스트 파일 예시:
  - PDF 스캔본 또는 출판사 PDF
  - HWP/HWPX/DOCX 원본
  - HWP/DOCX/PDF가 섞인 ZIP
  - 문제 이미지 1~3개

## 2. 합격 기준

| 영역 | 합격 기준 |
|------|-----------|
| 업로드 | PDF/HWP/HWPX/DOCX/ZIP/이미지를 같은 화면에서 등록할 수 있다. |
| 다운로드 | ZIP이 다운로드되고, `00_먼저열기_검수체크리스트.doc`, `01_자체양식_문제검수본.doc`, `02_OCR_연결후보.csv`가 포함된다. |
| 열림 | 산출 `.doc` 파일이 한글 또는 Word에서 열린다. |
| 원본 보존 | PDF/이미지 페이지와 HWP 임베디드 이미지가 깨지지 않는다. |
| 구조화 | `00_manifest.json`에 구조화 항목 수, 문제 후보 수, OCR 후보 수, 품질 상태가 기록된다. |
| 수정성 | 선생님이 본문/문항/안내문을 직접 고칠 수 있다. |
| 경고 | 제한된 파일은 조용히 실패하지 않고 체크리스트/리포트에 기록된다. |
| Beta | 재작성은 접힌 Beta 패널에 남고, 기본 이관을 방해하지 않는다. |

## 3. 기본 UAT 절차

1. Problem Studio에 접속한다.
2. `소스 업로드`로 실제 원본 파일을 모두 등록한다.
3. 제목/반명/과목을 실제 수업명으로 입력한다.
4. `원본 한글로 저장`을 누른다.
5. ZIP을 열고 `00_먼저열기_검수체크리스트.doc`를 먼저 연다.
6. `01_자체양식_문제검수본.doc`에서 자동 분리된 문제/개념 후보를 확인한다.
7. `02_OCR_연결후보.csv`에서 OCR 후보 ID, 원본, 쪽 범위, 우선순위, 권장처리를 확인한다.
8. `00_변환리포트.html`에서 문서 수, 이미지 수, 경고 수, 구조화 상태, OCR 후보 수를 확인한다.
9. `00_manifest.json`에서 `quality_level`, `structured_problem_count`, `ocr_candidate_count`를 확인한다.
10. 각 산출 `.doc`을 한글 또는 Word에서 열어 첫 쪽/중간 쪽/마지막 쪽을 원본과 대조한다.
11. 표, 수식, 그림, 선택지, 정답/해설이 빠지거나 섞인 부분을 체크리스트의 피드백 표에 적는다.
12. 수정 후 수업용 파일로 저장한다.

## 4. 파일 유형별 확인 포인트

| 유형 | 확인 |
|------|------|
| PDF | 페이지 수가 맞는지, 잘림/회전/흐림이 없는지 확인한다. 스캔본은 이미지 보존이 합격 기준이다. |
| HWP | 본문 텍스트, 임베디드 그림/도표, 표·수식 위치를 원본과 대조한다. |
| HWPX/DOCX | 본문 텍스트 누락이 없는지 확인한다. 표/이미지는 원본 대조가 필요하다. |
| ZIP | 내부 지원 파일이 모두 산출물로 나왔는지 `00_파일목록.csv`와 대조한다. |
| 이미지 | 해상도와 잘림을 확인한다. OCR 전까지 텍스트 편집은 수동 기준이다. |
| DOC | 경고가 떠야 정상이다. DOCX 또는 PDF로 저장 후 재업로드하는 회피 경로를 안내한다. |

## 5. 파괴 테스트

| 케이스 | 기대 결과 |
|--------|-----------|
| 지원하지 않는 확장자만 있는 ZIP | 지원 파일 없음 경고가 리포트에 기록된다. |
| 내부 파일 수가 제한을 넘는 ZIP | ZIP 해제 오류 문서 1개와 경고가 생성된다. |
| 깨진 PDF/HWP/DOCX | 해당 원본의 오류 리포트가 생성되고 전체 요청은 가능한 범위에서 완료된다. |
| 큰 PDF | 60쪽 단위로 `.doc`이 분할된다. |
| 이미지-only PDF | 페이지 이미지는 보존되고, manifest/report/checklist/`02_OCR_연결후보.csv`에 OCR 후보로 기록된다. |

## 6. 구조화 검수

`01_자체양식_문제검수본.doc`는 최종 교재가 아니라 빠른 검수본이다.

1. 상단의 자동 분리 항목 수, 문제 후보 수, 개념 블록 수를 확인한다.
2. 번호가 잘못 끊긴 항목, 표/수식이 본문으로 풀린 항목, 선택지가 누락된 항목을 표시한다.
3. `정답: 검수 필요`, `해설: 검수 후 작성` 항목은 선생님이 직접 채운다.
4. OCR 후보가 있으면 `02_OCR_연결후보.csv`의 후보 ID와 쪽 범위를 기준으로 시각 보존 합격/수동 전사/비동기 OCR 보강 여부를 결정한다.

## 7. Beta 재작성 검수

Beta는 상품의 기본 약속이 아니다. 아래 기준으로만 확인한다.

1. Beta 패널은 기본 접힘 상태여야 한다.
2. `유사 유형`, `함정 보강`, `개념형` 중 하나를 선택한다.
3. 후보 수를 1~10 사이로 둔다.
4. 결과는 반드시 선생님 검수용 초안으로만 취급한다.
5. 정답/해설이 불확실하면 그대로 배포하지 않는다.

## 8. 피드백 수집 양식

선생님에게 아래 5가지만 받는다.

| 질문 | 답변 |
|------|------|
| 원본을 다시 타이핑하는 시간 대비 얼마나 줄었는가? | |
| 한글에서 바로 열고 수정할 수 있었는가? | |
| 누락/깨짐이 발생한 파일명과 위치는 어디인가? | |
| 결과물에서 가장 먼저 고치고 싶은 불편은 무엇인가? | |
| 유료 기능이라면 OCR, HWPX 출력, 문제 분리 중 무엇을 먼저 원하나? | |

## 9. 다음 개선 판단

- 스캔본/이미지 자료가 많고 텍스트 수정 요구가 높으면 OCR을 우선한다.
- 한글 원본 양식 재현 요구가 높으면 native HWPX 출력과 템플릿 렌더러를 우선한다.
- 선생님이 내용 재구성을 원하면 Beta 후보 승인/거절 흐름을 먼저 만든다.
