# 메시지 발송 실패: 솔라피(SOLAPI) IP 허용 설정

**증상:** 학생 관리 등에서 "메시지 발송"을 했을 때 발송 내역(메시지 > 발송 내역)에는 실패 건으로 기록되지만, 실제로 문자가 나가지 않음.

**비고에 보이는 에러 예:**
- `(Forbidden', '허용되지 않은 IP(54.180.247.234)로 접근하고 있습니다.')`
- 또는 `솔라피 IP 미등록: SOLAPI 콘솔...` (워커에서 안내 문구를 붙인 경우)

---

## 원인

SOLAPI(솔라피)에서 **API 호출 허용 IP**를 제한해 두었을 때, 학원 서버(또는 메시지 워커)의 **나가는 IP**가 그 목록에 없으면 요청이 거절됩니다.  
→ 발송 요청은 큐에 들어가고, 워커가 Solapi를 호출한 시점에 거절되어 발송 내역에는 “실패”로만 남습니다.

---

## 해결 방법

1. **SOLAPI 콘솔** 접속: [https://console.solapi.com](https://console.solapi.com)
2. **설정** 메뉴에서 **허용 IP** (또는 API 보안/IP 제한) 항목을 찾습니다.
3. 비고에 표시된 **IP 주소**(예: `54.180.247.234`)를 **허용 목록에 추가**합니다.
4. 실제 발송을 수행하는 서버/워커가 다른 IP로 나간다면, 해당 **나가는 IP**를 확인한 뒤 그 IP를 등록합니다.
   - AWS EC2 등에서 나가는 IP 확인: 인스턴스에서 `curl -s ifconfig.me` 또는 NAT Gateway의 Elastic IP를 확인합니다.

이후 같은 환경에서의 발송은 정상 처리될 가능성이 높습니다.  
계속 실패하면 SOLAPI 쪽 응답 메시지(비고에 기록된 문구)를 확인해 추가 설정이 필요한지 봅니다.
